## 一、面向对象核心概念 🧱

C++ 是面向对象的语言，程序 = 对象集合 + 交互

| 概念 | 通俗解释 | 示例（狗） |
|------|---------|-----------|
| **对象 (Object)** | 具有**状态**和**行为**的实体 | 一只具体的狗：名字叫"旺财"，会摇尾巴 |
| **类 (Class)** | 描述对象的**模板/蓝图** | `Dog` 类：定义狗有名字、颜色，能叫、能吃 |
| **方法 (Method)** | 对象能执行的**行为/功能** | `bark()`, `eat()`, `wagTail()` |
| **实例变量** | 每个对象独有的**状态数据** | `name="旺财"`, `color="黄色"`, `breed="柴犬"` |

```cpp
// 简单类比
类：汽车设计图纸
对象：根据图纸造出的具体汽车（红色宝马、白色特斯拉...）
方法：汽车能做的动作（启动、加速、刹车）
实例变量：每辆车的独特属性（颜色、车牌、里程）
```

## 二、C++ 程序结构解析 📄

### 2.1 Hello World 完整示例

```cpp
#include <iostream>        // ① 头文件：包含输入输出功能
using namespace std;       // ② 命名空间：避免名字冲突
 
// ③ 单行注释：以 // 开头，编译器忽略
 
int main()                 // ④ 主函数：程序入口，从这里开始执行
{
   cout << "Hello World";  // ⑤ 输出语句：在屏幕打印文字
   return 0;               // ⑥ 返回值：0 表示程序正常结束
}
```

### 2.2 关键要素详解

| 代码行 | 作用 | 注意事项 |
|--------|------|---------|
| `#include <iostream>` | 引入标准输入输出库 | 尖括号`<>`表示系统头文件 |
| `using namespace std;` | 使用标准命名空间 | 避免每次写 `std::cout` |
| `// 注释` | 添加说明，编译器忽略 | 单行`//`，多行`/* */` |
| `int main()` | 主函数，程序起点 | 必须有，且返回 `int` |
| `cout << "文本";` | 输出内容到屏幕 | `<<` 是流插入运算符 |
| `return 0;` | 结束函数，返回状态码 | 0=成功，非 0=异常 |

## 三、编译 & 执行流程 ⚙️

### 3.1 手动编译四步法

```bash
# 1️⃣ 编写代码：保存为 hello.cpp
# 2️⃣ 打开终端，进入文件所在目录
# 3️⃣ 编译代码（g++ 是 GNU C++ 编译器）
$ g++ hello.cpp

# 4️⃣ 运行生成的可执行文件
$ ./a.out          # Linux/Mac
$ a.out.exe        # Windows
Hello World        # 输出结果
```

### 3.2 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| `g++: command not found` | 未安装编译器 | 安装 MinGW (Win) 或 build-essential (Linux) |
| 编译报错 | 代码语法错误 | 检查分号、括号、拼写 |
| 运行无输出 | 忘记 `cout` 或缓冲区 | 添加 `endl` 或 `flush` |
| 中文乱码 | 编码不一致 | 保存为 UTF-8，终端设置匹配 |

### 3.3 进阶：使用 Makefile（可选）
```makefile
# Makefile 示例
hello: hello.cpp
	g++ hello.cpp -o hello

clean:
	rm -f hello
```
```bash
$ make      # 编译
$ ./hello   # 运行
$ make clean # 清理
```

## 四、基础语法规则 📏

### 4.1 分号与语句块

```cpp
// ✅ 每个语句必须以分号 ; 结尾
x = y;
y = y + 1;
add(x, y);

// ✅ 语句块：用 { } 包裹逻辑相关的多条语句
{
    cout << "开始" << endl;
    // ... 多条语句 ...
    cout << "结束" << endl;
}

// ✅ C++ 不强制换行，一行可写多句（但不推荐，影响可读性）
x = y; y = y+1; add(x, y);  // 语法正确，但难读 ❌
```

### 4.2 标识符命名规则

```cpp
// ✅ 有效标识符（变量/函数/类名）
mohd      zara      abc       move_name   a_123
myname50  _temp     j         a23b9       retVal

// ❌ 无效标识符
123abc    // 不能以数字开头
my-name   // 不能含连字符 -
@class    // 不能含特殊字符 @#$%^&*

// 📋 命名规则总结：
// 1. 首字符：字母 (A-Z/a-z) 或 下划线 _
// 2. 后续字符：字母、数字、下划线
// 3. 区分大小写：Manpower ≠ manpower
// 4. 不能是关键字（见下表）
```

### 4.3 C++ 关键字（保留字）⚠️

> 以下单词有特殊含义，**不能**用作变量名、函数名等标识符

```cpp
// 常用关键字分类记忆：

🔹 数据类型：int, float, double, char, bool, void
🔹 流程控制：if, else, for, while, do, switch, case, break, continue
🔹 函数相关：return, inline, virtual, static, const
🔹 内存管理：new, delete, malloc, free
🔹 面向对象：class, struct, public, private, protected, this
🔹 异常处理：try, catch, throw
🔹 类型转换：static_cast, dynamic_cast, const_cast, reinterpret_cast
🔹 命名空间：namespace, using, typedef, typename
🔹 其他：asm, auto, extern, register, sizeof, template, true, false

// 💡 建议：IDE 通常会对关键字高亮，写代码时注意避开
```

### 4.4 三字符组（Trigraphs）🔍

```cpp
// 三字符组：用 ?? 开头的 3 字符序列，代表单个特殊字符
// 历史原因：早期键盘缺少某些符号，现基本不用

??=  →  #     ??/  →  \     ??'  →  ^
??(  →  [     ??)  →  ]     ??!  →  |
??<  →  {     ??>  →  }     ??-  →  ~

// ⚠️ 注意：
// 1. 现代编译器（如 VS2010+）默认禁用三字符组
// 2. g++ 仍支持但会警告
// 3. 如果想写两个问号 ??，可用转义：?\? 或字符串拼接 "?""?"
```

### 4.5 空格与可读性

```cpp
// ✅ 编译器忽略多余空格，但人类需要可读性

// 必须有空格的情况：
int age;  // int 和 age 之间至少一个空格，否则变成 intage

// 可选但推荐加空格：
fruit = apples + oranges;  // 运算符两侧加空格，更易读

// 空白行 + 注释：提升代码结构清晰度
int score;     // 学生分数

double price;  // 商品单价
```

## 五、C++ 注释写法 📝

```cpp
// ① 单行注释：以 // 开头，到行末结束
cout << "Hello";  // 输出问候语

/* ② 多行注释：以 /* 开头，*/ 结尾
   可以跨越多行
   常用于函数说明、版权信息等
*/

// ③ 文档注释（Doxygen 风格，可选）
/**
 * 计算两数之和
 * @param a 第一个数
 * @param b 第二个数
 * @return 两数之和
 */
int add(int a, int b) {
    return a + b;
}

// 💡 最佳实践：
// • 复杂逻辑必加注释
// • 注释说明"为什么"而非"是什么"
// • 保持注释与代码同步更新
```

## 六、新手避坑指南 ⚠️

| 常见错误 | 正确写法 | 说明 |
|---------|---------|------|
| 忘记分号 | `cout << "Hi";` | 每条语句末尾加分号 |
| 主函数写错 | `int main()` | 必须是 `int main`，不能漏 `int` |
| 头文件错误 | `#include <iostream>` | 尖括号`<>` vs 双引号`""` |
| 命名冲突 | 避免用 `class`, `new` 等关键字 | 关键字不能作标识符 |
| 中文编码 | 保存为 UTF-8 无 BOM | 避免中文注释乱码 |
| 路径问题 | `g++ hello.cpp` 在当前目录 | 确保终端在正确路径 |
