# C++ 听课笔记 - 变量类型 (Variable Types)

> 📌 **来源**：菜鸟教程 C++ 教程  
> 🎯 **目标**：掌握 C++ 变量的定义、声明、初始化与核心概念  
> 💡 **核心思想**：变量 = 有名字的内存空间，类型 = 空间的"使用规则"

---

## 一、变量的本质 🧠

```cpp
/*
 * 🔍 变量是什么？
 * 
 * 变量 = 程序可操作的【存储区名称】
 * 
 * 类比理解：
 * 📦 内存 = 仓库
 * 🔖 变量名 = 货架标签  
 * 📋 数据类型 = 货架规格（能放什么、放多少）
 * 📦 变量值 = 货架上的货物
 * 
 * ✅ 变量的三要素：
 * ① 名称（标识符）：如何找到它
 * ② 类型：能存什么、占多大
 * ③ 值：当前存储的具体数据
 */
```

## 二、变量命名规则 🏷️

### 2.1 合法标识符规范

```cpp
// ✅ 命名规则：
// 1. 字符组成：字母 (A-Z/a-z)、数字 (0-9)、下划线 (_)
// 2. 首字符：必须是字母或下划线（不能是数字！）
// 3. 区分大小写：Age ≠ age ≠ AGE
// 4. 不能是关键字：int, class, return 等保留字不能用

// ✅ 有效示例：
int age;
double salary;
char _temp;
std::string userName;      // 驼峰命名
std::vector<int> data_list; // 下划线命名

// ❌ 无效示例：
int 123abc;    // ❌ 不能以数字开头
float my-name; // ❌ 不能含连字符 -
char @symbol;  // ❌ 不能含特殊字符 @#$%
int class;     // ❌ 不能是关键字
```

### 2.2 命名风格建议（最佳实践）

```cpp
// 🎯 推荐风格（二选一，保持项目统一）：

// 🔹 驼峰命名法 (camelCase) - 变量/函数
int studentCount;
double totalPrice;
void calculateAverage();

// 🔹 下划线命名法 (snake_case) - 变量/函数
int student_count;
double total_price;
void calculate_average();

// 🔹 大驼峰 (PascalCase) - 类/结构体
class StudentRecord;
struct UserInfo;

// 🔹 全大写 + 下划线 - 常量/宏
const int MAX_SIZE = 100;
#define PI 3.14159

// 💡 原则：见名知意，避免 a, b, c 这类无意义命名
```

## 三、变量类型速查表 📊

### 3.1 基本类型回顾

| 类型 | 描述 | 典型大小 | 取值示例 |
|------|------|---------|---------|
| `bool` | 布尔值 | 1 字节 | `true` / `false` |
| `char` | ASCII 字符 | 1 字节 | `'A'`, `'z'`, `65` |
| `int` | 整数 | 4 字节 | `-100`, `0`, `2024` |
| `float` | 单精度浮点数 | 4 字节 | `3.14f`, `-0.001f` |
| `double` | 双精度浮点数 | 8 字节 | `2.718281828`, `1.23e-4` |
| `void` | 无类型 | - | 用于函数无返回值 |
| `wchar_t` | 宽字符 | 2/4 字节 | `L'中'`, `L'文'` |

### 3.2 扩展类型分类

```cpp
// 🔹 整数类型族
short          // 短整型，通常 2 字节
int            // 普通整型，通常 4 字节  
long           // 长整型，通常 4 或 8 字节
long long      // 长长整型，通常 8 字节（C++11 保证）

// 🔹 浮点类型族
float          // 单精度，6-7 位有效数字
double         // 双精度，15 位有效数字
long double    // 扩展精度，平台相关

// 🔹 字符类型族
char           // 8 位 ASCII
wchar_t        // 宽字符，平台相关
char16_t       // UTF-16，2 字节（C++11）
char32_t       // UTF-32，4 字节（C++11）

// 🔹 派生类型（后续章节详解）
enum Color { RED, GREEN };     // 枚举
int* ptr = &x;                 // 指针
int& ref = x;                  // 引用
int arr[10];                   // 数组
struct Point { int x, y; };    // 结构体
class Student { ... };         // 类
union Data { int i; float f; }; // 联合体
```

> ⚠️ **重要提示**：类型大小与编译器、操作系统、CPU 架构相关，不确定时用 `sizeof(类型)` 查询！

## 四、变量定义与初始化 ✍️

### 4.1 变量定义语法

```cpp
// 🔹 基本语法：type variable_list;
int i, j, k;              // 定义 3 个 int 变量
char c, ch;               // 定义 2 个 char 变量
double salary, bonus;     // 定义 2 个 double 变量

// 🔹 定义 + 初始化：type variable_name = value;
int age = 18;                      // 直接初始化
double price = 19.99, tax = 0.08;  // 多个变量同时初始化
char grade = 'A';                  // 字符用单引号
bool isPassed = true;              // 布尔值
```

### 4.2 初始化方式对比

```cpp
// 🔹 C 风格初始化（传统）
int x = 10;
double y(3.14);        // 构造函数风格

// 🔹 C++11 统一初始化（推荐 ✅）
int a{10};             // 花括号初始化
double b{3.14};
std::vector<int> v{1, 2, 3};  // 容器也适用

// 💡 统一初始化的优势：
// • 语法一致，避免歧义
// • 防止窄化转换（narrowing）
int narrow{3.14};  // ❌ 编译错误：double → int 可能丢失精度

// 🔹 默认初始化（不指定初始值）
int uninit;          // ⚠️ 局部变量：值是未定义的（垃圾值）
static int s;        // ✅ 静态变量：自动初始化为 0
global int g;        // ✅ 全局变量：自动初始化为 0
```

### 4.3 初始化最佳实践

```cpp
// ✅ 推荐：定义时立即初始化
int count = 0;
std::string name = "Alice";

// ❌ 避免：使用未初始化的变量
int score;
// ... 很多行代码后 ...
if (score > 60) { ... }  // ⚠️ score 可能是随机值！

// ✅ 特殊情况：后续赋值的变量
int result;  // 先声明
// ... 复杂计算 ...
result = calculate();  // 再赋值（确保赋值前不使用）
```

## 五、变量声明 vs 定义 🔍

### 5.1 核心区别

| 特性 | 声明 (Declaration) | 定义 (Definition) |
|------|------------------|------------------|
| **作用** | 告诉编译器"有这个变量" | 告诉编译器"创建这个变量" |
| **内存分配** | ❌ 不分配 | ✅ 分配存储空间 |
| **次数限制** | 可多次声明 | 只能定义一次 |
| **关键字** | 常用 `extern` | 直接写类型 + 变量名 |
| **时机** | 编译时检查 | 链接时确定地址 |

### 5.2 extern 关键字详解

```cpp
// 📁 场景：多文件项目，变量在 file1.cpp 定义，在 file2.cpp 使用

// file1.cpp（定义变量 - 只出现一次）
int globalCount = 0;  // ✅ 定义 + 初始化

// file2.cpp（声明变量 - 可多次声明）
extern int globalCount;  // ✅ 声明：告诉编译器 globalCount 在其他地方定义

void increment() {
    globalCount++;  // ✅ 可以使用
}

// 🔹 声明 vs 定义 的关键区别：
extern int x;      // ✅ 纯声明：不分配内存
extern int y = 10; // ⚠️ 这其实是定义！有初始化就是定义

// 💡 记忆口诀：
// "extern + 无初始化 = 声明"
// "有初始化 或 无 extern = 定义"
```

### 5.3 函数声明 vs 定义（类比理解）

```cpp
// 🔹 函数声明（原型）：告诉编译器函数签名
int add(int a, int b);  // ✅ 声明：返回类型 + 函数名 + 参数

// 🔹 函数定义：提供具体实现
int add(int a, int b) {  // ✅ 定义：有函数体
    return a + b;
}

// 🔹 使用顺序：先声明，后调用，定义可在任何位置
int main() {
    int result = add(3, 4);  // ✅ 调用（编译器已知 add 的签名）
    return 0;
}
// add 的定义可以在 main 之后
```

## 六、左值 (Lvalue) 与右值 (Rvalue) ⚖️

### 6.1 核心概念

```cpp
/*
 * 🔍 左值 (lvalue)：
 * • 有内存地址的表达式
 * • 可以出现在赋值号左边（可被赋值）
 * • 通常是：变量、解引用指针、函数返回引用
 * 
 * 🔍 右值 (rvalue)：
 * • 临时值/字面量/表达式结果
 * • 只能出现在赋值号右边（不可被赋值）
 * • 通常是：数字、字符串字面量、临时对象
 */
```

### 6.2 实例对比

```cpp
// ✅ 左值示例（有地址，可赋值）
int x = 10;        // x 是左值
x = 20;            // ✅ 左值在左边：合法

int arr[5];
arr[0] = 100;      // ✅ arr[0] 是左值

int& ref = x;
ref = 30;          // ✅ 引用是左值

// ✅ 右值示例（无地址，不可赋值）
10 = x;            // ❌ 错误：字面量 10 是右值，不能在左边
x + 1 = 5;         // ❌ 错误：表达式 x+1 是右值
"hello" = str;     // ❌ 错误：字符串字面量是右值

// 🔹 特殊情况：const 左值
const int c = 100;
// c = 200;        // ❌ 错误：c 是 const 左值，不可修改

// 🔹 现代 C++：右值引用 (C++11)
int&& rref = 42;   // ✅ rref 是右值引用，绑定到右值 42
```

### 6.3 快速判断技巧

```cpp
// 🎯 判断表达式是左值还是右值：
// 问：这个表达式能取地址吗？(&expr 合法吗？)

int x = 10;
&x;        // ✅ 合法 → x 是左值

&(x + 1);  // ❌ 非法 → x+1 是右值

// 🎯 判断能否出现在赋值号左边：
x = 20;        // ✅ x 是左值，可被赋值
(x + 1) = 30;  // ❌ x+1 是右值，不可被赋值
```

## 七、实用技巧与避坑指南 🛠️

### 7.1 类型选择建议

```cpp
// 🔹 整数选择：
int count = 100;              // 普通计数
unsigned int id = 12345U;     // 只存非负数（注意溢出行为！）
long long bigNum = 1LL << 40; // 大整数
int8_t small = -50;           // 精确控制大小（需 #include <cstdint>）

// 🔹 浮点数选择：
float price = 19.99f;         // 内存敏感/图形计算
double precise = 3.14159265;  // 一般科学计算（默认选 double）

// 🔹 字符选择：
char ascii = 'A';             // ASCII 文本
wchar_t wide = L'中';         // 宽字符（平台相关）
char32_t unicode = U'😀';     // UTF-32（C++11，跨平台）

// 🔹 布尔选择：
bool isActive = true;         // 逻辑判断（比 int 更语义化）
```

### 7.2 常见陷阱 ⚠️

```cpp
// ❌ 陷阱 1：未初始化局部变量
int score;  // ⚠️ 值是随机的！
if (score > 60) { ... }  // 行为未定义

// ✅ 正确：始终初始化
int score = 0;

// ❌ 陷阱 2：整数溢出（尤其 unsigned）
unsigned int u = 0;
u = u - 1;  // ⚠️ 不是 -1！而是 4294967295（回绕）

// ✅ 正确：检查边界或使用有符号类型
if (u > 0) u--;

// ❌ 陷阱 3：浮点数比较用 ==
double a = 0.1 + 0.2;
if (a == 0.3) { ... }  // ⚠️ 可能为 false！浮点误差

// ✅ 正确：用误差范围比较
#include <cmath>
if (std::abs(a - 0.3) < 1e-9) { ... }

// ❌ 陷阱 4：char 当整数用时的符号问题
char c = 200;  // ⚠️ 如果 char 是有符号的，200 会变成负数！
// ✅ 正确：明确使用 unsigned char
unsigned char uc = 200;
```

### 7.3 调试技巧

```cpp
// 🔹 打印变量类型信息（C++11）
#include <typeinfo>
#include <iostream>

int x = 10;
std::cout << "x 的类型: " << typeid(x).name() << std::endl;

// 🔹 打印变量地址（调试指针/引用）
int y = 20;
int* ptr = &y;
std::cout << "y 的值: " << y << ", 地址: " << &y << std::endl;
std::cout << "ptr 的值: " << ptr << ", ptr 的地址: " << &ptr << std::endl;

// 🔹 使用 static_assert 编译时检查类型
static_assert(sizeof(int) == 4, "int 必须是 4 字节！");
```

## 八、核心知识速查表 ✅

```cpp
🔹 变量三要素：名称 + 类型 + 值
🔹 命名规则：字母/下划线开头，含字母/数字/下划线，区分大小写
🔹 定义 vs 声明：
   • 定义：分配内存，只能一次
   • 声明：extern 关键字，可多次，不分配内存
🔹 初始化：
   • 局部变量：不初始化 = 垃圾值 ⚠️
   • 全局/静态变量：自动初始化为 0 ✅
   • 推荐：定义时立即初始化
🔹 左值 vs 右值：
   • 左值：有地址，可赋值（变量、引用）
   • 右值：无地址，不可赋值（字面量、临时值）
🔹 类型选择原则：
   • 整数：int 默认，大数用 long long，精确控制用 <cstdint>
   • 浮点：double 默认，内存敏感用 float
   • 字符：char 默认，Unicode 用 char16_t/char32_t
🔹 避坑要点：
   • 始终初始化局部变量
   • unsigned 注意溢出回绕
   • 浮点数比较用误差范围
   • char 注意有符号/无符号差异
```
