## 基本概念

### 什么是头文件（.h文件）

头文件是C++程序中用来**声明**函数、类、变量等的文件。它主要告诉编译器"有什么东西可用"。

**特点：**

- 文件扩展名通常是 `.h` 或 `.hpp`
- 只包含声明，不包含实现
- 可以被多个源文件包含
- 用 `#include` 指令包含

**例子：**

```cpp
// mystack.h - 头文件
class MyStack {
public:
    void push(int x);    // 只有声明，告诉编译器有这个函数
    void pop();
};
```

### 什么是源文件（.cpp文件）

源文件是包含**具体实现**代码的文件，它真正定义了函数和类的具体行为。

**特点：**

- 文件扩展名通常是 `.cpp` 或 `.cxx`
- 包含函数的具体实现代码
- 编译成机器码
- 包含对应的头文件

**例子：**

```cpp
// mystack.cpp - 源文件
#include "mystack.h"

void MyStack::push(int x) {    // 具体实现，函数真正做什么
    // 实际的代码
}
```

## 为什么要分离

### 1. **模块化和代码组织**

将相关功能组织在一起，便于管理和维护。

**对比：**

```
不分离（全部写在一个文件）：
├── main.cpp (2000行代码，所有功能混在一起)

分离（模块化）：
├── main.cpp        (主程序，50行)
├── stack.h        (栈的声明，30行)
├── stack.cpp      (栈的实现，200行)
├── calculator.h   (计算器的声明，25行)
└── calculator.cpp (计算器的实现，150行)
```

### 2. **提高编译效率**

修改一个文件时，只需要重新编译这个文件，其他文件不需要重新编译。

**示例：**

```cpp
// main.cpp
#include "stack.h"
#include "calculator.h"

int main() {
    MyStack s;
    Calculator c;
}
```

当你修改 `stack.cpp` 时：

- ✅ 只需要重新编译 `stack.cpp`
- ✅ 不需要重新编译 `calculator.cpp`
- ✅ 不需要重新编译 `main.cpp`（因为它只包含头文件）

### 3. **代码重用**

头文件可以被多个源文件包含，避免重复代码。

**例子：**

```cpp
// utilities.h
void printHello();
void printGoodbye();

// file1.cpp
#include "utilities.h"
void function1() {
    printHello();    // 使用utilities中的函数
}

// file2.cpp
#include "utilities.h"
void function2() {
    printHello();    // 也可以使用utilities中的函数
}
```

### 4. **接口和实现分离**

头文件定义"接口"（公开的功能），源文件定义"实现"（具体的做法）。

**类比：**

```
头文件 = 产品说明书（告诉你产品能做什么）
源文件 = 产品内部构造（如何实现这些功能）
```

## 它们之间的关系

### 1. **声明与定义的关系**

**头文件中的声明：**

```cpp
// mystack.h
class MyStack {
public:
    void push(int x);     // 声明：告诉编译器有这个函数
};
```

**源文件中的定义：**

```cpp
// mystack.cpp
#include "mystack.h"

void MyStack::push(int x) {    // 定义：函数的具体实现
    // 实际代码
}
```

### 2. **包含关系**

源文件必须包含对应的头文件：

```cpp
// mystack.cpp
#include "mystack.h"    // 必须包含，这样才能使用MyStack类

void MyStack::push(int x) {
    // 实现
}
```

使用这些类的文件也要包含头文件：

```cpp
// main.cpp
#include "mystack.h"    // 包含头文件才能使用MyStack

int main() {
    MyStack stack;    // 现在可以使用MyStack了
    stack.push(10);
}
```

### 3. **一对一或一对多关系**

**典型的一对一关系：**

```
mystack.h  ←→  mystack.cpp
calculator.h  ←→  calculator.cpp
```

**一对多关系（一个头文件对应多个源文件）：**

```
bigclass.h  ←→  bigclass_part1.cpp
           ←→  bigclass_part2.cpp
           ←→  bigclass_part3.cpp
```

## 编译和链接过程

### 完整的编译流程：

```
1. 预处理阶段
   main.cpp  →  预处理器  →  main.i
                    ↓
                #include "mystack.h" 被展开

2. 编译阶段
   main.i    →  编译器    →  main.obj (目标文件)
   mystack.cpp →  编译器    →  mystack.obj

3. 链接阶段
   main.obj + mystack.obj →  链接器 →  program.exe (可执行文件)
```

### 详细步骤解释：

**步骤1：预处理**

```cpp
// main.cpp
#include "mystack.h"

int main() {
    MyStack s;
}
```

预处理后，`#include` 被替换为头文件的实际内容：

```cpp
// main.i (预处理后的文件)
// mystack.h 的所有内容都在这里

int main() {
    MyStack s;    // 现在编译器知道MyStack是什么了
}
```

**步骤2：编译**
每个 `.cpp` 文件被独立编译成 `.obj` 文件：

```
main.cpp    →  main.obj
mystack.cpp →  mystack.obj
```

**步骤3：链接**
链接器将所有 `.obj` 文件组合成一个可执行文件：

```
main.obj + mystack.obj →  program.exe
```

## 实际例子

### 示例1：简单的计算器

**头文件：calculator.h**

```cpp
#ifndef CALCULATOR_H
#define CALCULATOR_H

class Calculator {
public:
    double add(double a, double b);
    double subtract(double a, double b);
    double multiply(double a, double b);
    double divide(double a, double b);
};

#endif
```

**源文件：calculator.cpp**

```cpp
#include "calculator.h"

double Calculator::add(double a, double b) {
    return a + b;
}

double Calculator::subtract(double a, double b) {
    return a - b;
}

double Calculator::multiply(double a, double b) {
    return a * b;
}

double Calculator::divide(double a, double b) {
    if (b == 0) return 0;  // 简单处理除零错误
    return a / b;
}
```

**主文件：main.cpp**

```cpp
#include <iostream>
#include "calculator.h"

using namespace std;

int main() {
    Calculator calc;

    cout << "5 + 3 = " << calc.add(5, 3) << endl;
    cout << "5 - 3 = " << calc.subtract(5, 3) << endl;
    cout << "5 * 3 = " << calc.multiply(5, 3) << endl;
    cout << "5 / 3 = " << calc.divide(5, 3) << endl;

    return 0;
}
```

### 示例2：你的波兰计算器项目

**文件关系图：**

```
main.cpp
  ├─ #include "mystackfile.h"    (使用栈)
  └─ #include "utility.h"       (使用工具函数)

mystackfile.h    (栈的声明)
mystackfile.cpp   (栈的实现)
  └─ #include "mystackfile.h"

utility.h        (工具函数的声明)
utility.cpp      (工具函数的实现)
  └─ #include "utility.h"
```

**为什么这样设计：**

- `main.cpp` 只需要知道栈"能做什么"，不需要知道"怎么做"
- 如果栈的实现改变了，`main.cpp` 不需要修改
- 可以单独测试栈的功能

## 常见问题

### Q1: 为什么用 `#include "filename.h"` 而不是 `#include <filename.h>`？

**双引号 `""`：**

```cpp
#include "mystack.h"    // 在当前目录和指定目录中查找
```

- 用于自己写的头文件
- 先在当前目录查找
- 用于项目内部文件

**尖括号 `<>`：**

```cpp
#include <iostream>    // 在系统目录中查找
```

- 用于标准库头文件
- 只在系统目录查找
- 用于标准库和第三方库

### Q2: 什么是头文件保护？

```cpp
#ifndef MYSTACK_H_INCLUDED
#define MYSTACK_H_INCLUDED

// 头文件内容

#endif // MYSTACK_H_INCLUDED
```

**作用：**

- 防止同一个头文件被包含多次
- 避免重复定义错误

**为什么需要：**

```cpp
// main.cpp
#include "mystack.h"
#include "calculator.h"  // 如果calculator.h也包含了mystack.h
// 如果没有保护，mystack.h会被包含两次，导致错误
```

### Q3: 模板类为什么要包含 `.cpp` 文件？

```cpp
// main.cpp
#include "mystack.cpp"  // 模板类需要包含.cpp文件
```

**原因：**

- 模板的编译需要具体的类型信息
- 编译器在编译时生成模板实例
- 如果不包含 `.cpp`，链接时会找不到实现

### Q4: 改头文件需要重新编译吗？

**看情况：**

**不需要重新编译的情况：**

- 只修改了注释
- 只修改了格式

**需要重新编译的情况：**

- 修改了函数声明
- 修改了类定义
- 修改了公共接口

**命令行编译示例：**

```bash
# 修改了mystack.h后
g++ -c main.cpp      # 需要重新编译
g++ -c mystack.cpp   # 可能需要重新编译
g++ main.o mystack.o -o program  # 重新链接
```

### Q5: 可以在头文件中写函数实现吗？

**可以，但有注意事项：**

**适合在头文件中的情况：**

```cpp
// mystack.h
class MyStack {
public:
    inline bool empty() const {  // 内联函数
        return count == 0;
    }
};
```

**不适合在头文件中的情况：**

```cpp
// mystack.h
class MyStack {
public:
    void push(int x) {  // 复杂函数不应该在头文件中
        // 大量代码...
    }
};
```

**最佳实践：**

- 简单函数（如getter/setter）可以在头文件中
- 复杂函数应该在 `.cpp` 文件中

## 总结

### 头文件和源文件的核心关系：

| 方面         | 头文件 (.h)  | 源文件 (.cpp)  |
| ------------ | ------------ | -------------- |
| **内容**     | 声明         | 定义/实现      |
| **作用**     | 告诉"有什么" | 告诉"怎么做"   |
| **大小**     | 通常较小     | 可能很大       |
| **编译结果** | 不编译       | 编译成目标文件 |
| **可被包含** | 可以多次     | 通常只编译一次 |

### 使用原则：

1. **一个类通常对应一对文件**

   ```
   ClassName.h  ←→  ClassName.cpp
   ```

2. **头文件包含接口，源文件包含实现**

   ```cpp
   // .h: void function();     (声明)
   // .cpp: void function() {...} (定义)
   ```

3. **使用时包含头文件，编译所有源文件**

   ```bash
   g++ main.cpp mystack.cpp calculator.cpp -o program
   ```

4. **修改实现只重新编译对应的.cpp**
   ```bash
   # 修改了mystack.cpp后
   g++ -c mystack.cpp
   g++ main.o mystack.o calculator.o -o program
   ```

### 最佳实践建议：

✅ **推荐做法：**

- 每个类都有自己的头文件和源文件
- 头文件有保护（`#ifndef`）
- 头文件只包含声明
- 头文件包含其他头文件用 `""`
- 标准库头文件用 `<>`

❌ **避免做法：**

- 在头文件中写复杂函数实现
- 没有头文件保护
- 在源文件中包含 `.cpp` 文件（模板类除外）
- 在头文件中定义全局变量
