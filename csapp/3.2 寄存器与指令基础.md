# 听课笔记：寄存器与指令基础

## 1. 寄存器概述

### 1.1 寄存器的发展历程

| 处理器 | 位数 | 通用寄存器数量 | 特点                |
| ------ | ---- | -------------- | ------------------- |
| 8086   | 16位 | 8个            | 最早的处理器        |
| 386等  | 32位 | 8个（扩展）    | 寄存器位数扩展      |
| x86-64 | 64位 | 16个           | 原8个扩展 + 新增8个 |

### 1.2 寄存器的角色与编程规范

在程序中，不同寄存器扮演不同角色：

| 寄存器        | 功能                 |
| ------------- | -------------------- |
| `%rax`        | 保存函数返回值       |
| `%rsp`        | 保存程序栈的结束位置 |
| 6个参数寄存器 | 传递函数参数         |

> 了解寄存器的用法后，理解汇编代码会容易很多。

## 2. 指令的组成

### 2.1 操作码与操作数

大多数指令包含两部分：

```
指令格式：操作码 + 操作数
```

| 组成部分   | 说明                  | 示例                     |
| ---------- | --------------------- | ------------------------ |
| **操作码** | 决定CPU执行操作的类型 | `movq`、`addq`、`subq`   |
| **操作数** | 指令操作的数据        | 立即数、寄存器、内存引用 |

> 注意：`ret` 返回指令没有操作数

### 2.2 操作数的分类

| 类型         | 说明                                   | AT&T格式示例                 |
| ------------ | -------------------------------------- | ---------------------------- |
| **立即数**   | 以 `$` 开头，后跟整数（满足C语言定义） | `$5`、`$-3`                  |
| **寄存器**   | 64位、32位、16位、8位寄存器均可        | `%rax`、`%eax`、`%ax`、`%al` |
| **内存引用** | 根据地址访问内存                       | `(%rax)`                     |

---

## 3. 内存引用详解

### 3.1 内存抽象模型

- 将内存抽象成**字节数组**
- 存取数据需要：**起始地址** + **数据长度**

### 3.2 通用内存引用格式

```
Imm(rb, ri, s)
```

| 组成部分 | 名称       | 说明              |
| -------- | ---------- | ----------------- |
| `Imm`    | 立即数     | 偏移量            |
| `rb`     | 基址寄存器 | 基础地址          |
| `ri`     | 变址寄存器 | 数组索引相关      |
| `s`      | 比例因子   | 必须是 1、2、4、8 |

**有效地址计算公式：**

```
有效地址 = Imm + rb的值 + (ri的值 × s)
```

### 3.3 比例因子与数据类型

| 数据类型 | 比例因子 S | 原因  |
| -------- | ---------- | ----- |
| `char`   | 1          | 1字节 |
| `int`    | 4          | 4字节 |
| `double` | 8          | 8字节 |

> 编译器根据数组类型自动确定比例因子

### 3.4 内存引用的其他形式

| 格式          | 说明                | 等价形式       |
| ------------- | ------------------- | -------------- |
| `(rb)`        | 仅基址寄存器        | `0(rb)`        |
| `Imm(rb)`     | 立即数 + 基址寄存器 | -              |
| `(rb, ri)`    | 基址 + 变址         | `0(rb, ri, 1)` |
| `Imm(rb, ri)` | 三者组合            | -              |
| `(, ri, s)`   | 仅变址 + 比例因子   | `0(0, ri, s)`  |

> ⚠️ 注意区分：
>
> - 不带 `$` 的立即数 → 内存引用
> - 带括号的寄存器 → 内存引用

## 4. MOV 指令详解

### 4.1 MOV 指令家族

| 指令   | 操作数据大小 | 说明     |
| ------ | ------------ | -------- |
| `movb` | 1字节        | 字节传送 |
| `movw` | 2字节        | 字传送   |
| `movl` | 4字节        | 双字传送 |
| `movq` | 8字节        | 四字传送 |

### 4.2 操作数限制

| 源操作数 | 目的操作数 | 是否允许 |
| -------- | ---------- | -------- |
| 立即数   | 寄存器     | ✅       |
| 立即数   | 内存引用   | ✅       |
| 寄存器   | 寄存器     | ✅       |
| 寄存器   | 内存引用   | ✅       |
| 内存引用 | 寄存器     | ✅       |
| 内存引用 | 内存引用   | ❌       |
| 任意     | 立即数     | ❌       |

> **64位处理器限制**：`mov` 指令的源操作数和目的操作数**不能同时**是内存地址

### 4.3 内存到内存的传送

当需要将数据从内存一个位置复制到另一个位置时：

```asm
movq (%rsi), %rax    ; 第一步：从内存加载到寄存器
movq %rax, (%rdi)    ; 第二步：从寄存器写入内存
```

### 4.4 MOV 指令示例

```asm
movl $-1, %eax       ; 源：立即数，目的：32位寄存器
movb $-1, %al        ; 源：立即数，目的：8位寄存器
```

> 指令后缀必须与操作数大小匹配

### 4.5 特殊情况：64位立即数

| 指令      | 说明                                                   |
| --------- | ------------------------------------------------------ |
| `movq`    | 源操作数为立即数时，只能是32位补码，然后符号扩展到64位 |
| `movabsq` | 源操作数可以是任意64位立即数，目的操作数只能是寄存器   |

### 4.6 32位操作的副作用

**重要规则**：任何对寄存器生成32位值的指令，都会把该寄存器的**高位部分置为零**。

```asm
movabsq $0x0011223344556677, %rax   ; rax = 0x0011223344556677
movb    $-1, %al                    ; rax = 0x00112233445566FF（低8位变化）
movw    $-1, %ax                    ; rax = 0x001122334455FFFF（低16位变化）
movl    $-1, %eax                   ; rax = 0x00000000FFFFFFFF（高32位清零！）
```

## 5. 数据扩展传送指令

### 5.1 零扩展指令（MOVZ）

| 指令     | 源大小 | 目的大小 | 说明      |
| -------- | ------ | -------- | --------- |
| `movzbw` | 1字节  | 2字节    | 字节→字   |
| `movzbl` | 1字节  | 4字节    | 字节→双字 |
| `movzwl` | 2字节  | 4字节    | 字→双字   |
| `movzbq` | 1字节  | 8字节    | 字节→四字 |
| `movzwq` | 2字节  | 8字节    | 字→四字   |

> `Z` = Zero（零扩展）
> 后两个字符：第一个是源大小，第二个是目的大小

### 5.2 符号扩展指令（MOVS）

| 指令     | 源大小 | 目的大小 | 说明      |
| -------- | ------ | -------- | --------- |
| `movsbw` | 1字节  | 2字节    | 字节→字   |
| `movsbl` | 1字节  | 4字节    | 字节→双字 |
| `movswl` | 2字节  | 4字节    | 字→双字   |
| `movsbq` | 1字节  | 8字节    | 字节→四字 |
| `movswq` | 2字节  | 8字节    | 字→四字   |
| `movslq` | 4字节  | 8字节    | 双字→四字 |

> `S` = Sign（符号扩展）
> 符号扩展比零扩展多一条 `movslq`

### 5.3 为什么没有 movzlq？

因为 `movl` 指令本身就会把高32位置零，所以不需要专门的 `movzlq` 指令。

### 5.4 特殊指令：cltq

```asm
cltq    ; 无操作数
```

- **源操作数**：总是 `%eax`
- **目的操作数**：总是 `%rax`
- **效果**：等同于 `movslq %eax, %rax`
- **优势**：编码更紧凑

## 6. 小结

| 知识点     | 关键内容                         |
| ---------- | -------------------------------- |
| 寄存器     | 16个64位通用寄存器，各有特定用途 |
| 指令格式   | 操作码 + 操作数                  |
| 操作数类型 | 立即数、寄存器、内存引用         |
| 内存引用   | `Imm(rb, ri, s)` 通用格式        |
| MOV指令    | b/w/l/q 对应不同数据大小         |
| 扩展传送   | 零扩展(MOVZ)和符号扩展(MOVS)     |
