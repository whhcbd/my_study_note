# 听课笔记：浮点数的深入理解

## 1. 整数到浮点数的转换：以 12345 → 12345.0 为例

- **整数 12345 的二进制**（忽略高18位0，取低14位）：
  $$
  12345_{10} = [11000000111001]_2
  $$
- **规格化表示**：
  $$
  12345 = 1.1000000111001_2 \times 2^{13}
  $$
  其中小数点左移13位，隐含的1被保留。
- **构造单精度浮点数（32位）**：
  - **符号位 s**：0（正数）
  - **阶码字段 exp**：
    由 $E = e - bias$，$E = 13$，$bias = 127$，得 $e = 13 + 127 = 140$。
    140 的二进制：`10001100`（8位）。
  - **小数字段 frac**：
    丢弃隐含的1，取小数部分 `1000000111001`，共13位，后面补10个0，凑满23位。
  - 最终二进制：`0 10001100 10000001110010000000000`。

- **结论**：整数与浮点数的二进制在特定区域匹配，源于规格化表示中隐含的1和小数部分的直接对应。

## 2. 浮点数的舍入（Rounding）

### 2.1 为什么需要舍入？

- 浮点数只能近似表示实数，对于无法精确表示的值，需找到"最接近"的可表示值。

### 2.2 IEEE 754 定义的四种舍入方式

| 舍入方式       | 规则                                           | 示例（舍入到整数）         |
| -------------- | ---------------------------------------------- | -------------------------- |
| **向偶数舍入** | 舍入到最近的值，若在中间，则使最低有效位为偶数 | 1.4→1, 1.6→2, 1.5→2, 2.5→2 |
| **向零舍入**   | 正数向下，负数向上（向0方向）                  | 1.5→1, -1.5→-1             |
| **向下舍入**   | 向负无穷方向                                   | 1.5→1, -1.5→-2             |
| **向上舍入**   | 向正无穷方向                                   | 1.5→2, -1.5→-1             |

### 2.3 向偶数舍入的优势

- **避免统计偏差**：如果总是向上或向下舍入，大量计算的平均值会偏离真实值。向偶数舍入使得一半情况向上、一半向下，长期平均更准确。
- **适用于二进制**：最低有效位为0视为偶数，为1视为奇数。
  - 例：二进制 `10.11100`（精确到小数点后两位）：
    - 候选值：`10.11` 和 `11.00`。
    - 正好在中间，取最低有效位为偶数的 `11.00`。

## 3. 浮点运算的特性

### 3.1 加法不具有结合性

- 示例：
  ```c
  (3.14 + 1e10) - 1e10  → 0.0
  3.14 + (1e10 - 1e10)  → 3.14
  ```
- 原因：大数与小数相加时，小数可能因舍入而丢失。

### 3.2 乘法不具有结合性

- 示例：
  ```c
  (1e20 * 1e20) * 1e-20  → +∞
  1e20 * (1e20 * 1e-20)  → 1e20
  ```
- 原因：中间结果可能溢出或舍入。

### 3.3 乘法对加法不具有分配性

- 示例：
  ```c
  1e20 * (1e-20 - 1e-20)  → 0.0
  1e20 * 1e-20 - 1e20 * 1e-20  → NaN? 或 0.0（但可能因舍入产生微小误差）
  ```
- 实际中由于舍入，结果可能不完全相等。

### 3.4 启示

- 浮点运算不满足代数公理，编写科学计算或编译器时需特别注意。

## 4. C语言中浮点类型与整型的转换

| 转换方向               | 可能的问题                                       | 说明                                                      |
| ---------------------- | ------------------------------------------------ | --------------------------------------------------------- |
| **int → float**        | 不会溢出，但可能舍入                             | float 精度约7位十进制，int可能无法精确转换（如大整数）。  |
| **int/float → double** | 能保留精确值                                     | double 范围更大、精度更高。                               |
| **double → float**     | 可能溢出（值超出float范围）或舍入（精度损失）    | 若值在float范围内，可能因精度丢失而舍入。                 |
| **float/double → int** | 向零舍入（小数部分截断），可能溢出（如大浮点数） | 若浮点数超出int范围，结果未定义（常见为截断或饱和处理）。 |

- **示例**：

  ```c
  int i = 12345;
  float f = i;        // f = 12345.0（精确，因为12345可用23位小数表示）
  double d = i;       // d = 12345.0（精确）

  double big = 1e20;
  int j = big;        // 未定义行为（通常截断为最大/最小整数值，或实现定义）

  float pi_f = 3.1415926535;  // 可能舍入为 3.1415927（精度损失）
  ```

---

## 5. 总结

- **浮点数转换**：整数转浮点需规格化，涉及隐含1和指数偏置。
- **舍入**：向偶数舍入是最常用模式，能减少统计偏差。
- **运算特性**：浮点运算不满足结合律、分配律，需谨慎处理。
- **类型转换**：不同浮点/整型间转换可能引入溢出、舍入或未定义行为，编程时应明确意图并检查范围。
