# 听课笔记：有符号数与无符号数之间的转换

## 1. 强制类型转换
- **C语言允许数据类型间的强制转换**。
  - 示例：`short a = -12345; unsigned short b = (unsigned short) a;`
  - 结果：`b = 53191`（十进制）。
- **转换规则**：**位模式不变，解释这些位的方式改变**。
  - 将 `-12345` 的补码位模式直接解释为无符号数，得到 `53191`。

---

## 2. 数值关系推导
### 无符号数与有符号数的映射
- **B2U(X)** 与 **B2T(X)** 的差值：
  - 对于相同位模式 `X`，有：
    ```
    B2U(X) - B2T(X) = x_{w-1} * 2^w
    ```
    其中 `x_{w-1}` 是最高位（符号位）。
  - 移项得：
    ```
    B2U(X) = B2T(X) + x_{w-1} * 2^w
    ```

### 有符号数 → 无符号数（T2U）
- **T2U(x)** 函数：
  - 若 `x` 为非负数（`x_{w-1}=0`）：`T2U(x) = x`。
  - 若 `x` 为负数（`x_{w-1}=1`）：`T2U(x) = x + 2^w`。
  - 公式：
    ```
    T2U(x) = x + x_{w-1} * 2^w
    ```
    - 例如：`-12345`（16位）→ `-12345 + 2^16 = 53191`。

### 无符号数 → 有符号数（U2T）
- **U2T(u)** 函数：
  - 若 `u ≤ TMax`（最高位为0）：`U2T(u) = u`。
  - 若 `u > TMax`（最高位为1）：`U2T(u) = u - 2^w`。
  - 公式：
    ```
    U2T(u) = u - u_{w-1} * 2^w
    ```

---

## 3. C语言中的隐式转换
- **运算中若一个操作数为有符号，另一个为无符号，则有符号数会被隐式转换为无符号数**。
  - 示例：
    ```c
    if (-1 < 0U)  // 0U 是无符号数
        printf("true");
    else
        printf("false");
    ```
    - 输出：`false`。
    - 原因：`-1` 被转换为无符号数，成为 `UINT_MAX`（32位下为 `2^32 - 1`），比较的是 `4294967295 < 0`，显然不成立。
  - **陷阱**：这类隐式转换可能导致非直观的结果，编程时需谨慎。

---

## 4. 不同字长之间的转换
### 4.1 扩展（较小类型 → 较大类型）
- **无符号数扩展**：**零扩展**（填充0）。
  - 例如：`unsigned char a = 0x81`（8位）→ `unsigned short b = a`（16位），高位补0，得 `0x0081`，数值不变。
- **有符号数扩展**：**符号位扩展**（填充最高位的值）。
  - 非负数（最高位0）：补0。
  - 负数（最高位1）：补1。
  - 示例：`char a = -123`（8位，二进制 `10000101`）→ `short b = a`，高位补1，得 `1111111110000101`，数值仍为 -123。

#### 符号位扩展的数学证明
- 证明扩展1位后数值不变：
  - 设原数 `B2T_w([x_{w-1}, x_{w-2}, ..., x_0])`。
  - 扩展1位后：`B2T_{w+1}([x_{w-1}, x_{w-1}, x_{w-2}, ..., x_0])`。
  - 两者差值：
    ```
    B2T_{w+1} - B2T_w = x_{w-1} * (-2^w + 2^{w-1}) + Σ ... 化简后为 0。
    ```
  - 因此扩展任意位后数值不变。

### 4.2 截断（较大类型 → 较小类型）
- **截断规则**：丢弃高位，保留低位。
  - 例如：`int a = 0x12345678` → `short b = (short) a`，保留低16位 `0x5678`。
- **无符号数截断**：相当于取模运算。
  - 公式：截断到 `k` 位后的值 = 原值 `mod 2^k`。
  - 类比十进制：123456 截取后3位 456，即 123456 mod 1000。
- **有符号数截断**：
  - 步骤：
    1. 先将位模式解释为无符号数，进行截断（取模）。
    2. 再将截断后的无符号数解释为有符号数（U2T）。
  - 公式：
    ```
    B2T_k(x) = U2T_k( B2U_w(x) mod 2^k )
    ```

---

## 5. 关键点总结
- **转换不改变位模式，只改变解释方式**。
- **数值关系**：`T2U(x) = x + (x<0 ? 2^w : 0)`，`U2T(u) = u - (u≥2^{w-1} ? 2^w : 0)`。
- **隐式转换易导致逻辑错误**，特别是比较运算中。
- **扩展**：无符号零扩展，有符号符号位扩展。
- **截断**：无符号取模，有符号需经过两步转换。

---

## 6. 注意事项
- C语言标准未强制要求使用补码，但几乎所有机器采用补码，以上规则基于补码。
- 不同字长转换时，需明确目标类型的表示范围，避免溢出或符号变化。
- 编写可移植代码时，应避免依赖特定转换行为（如隐式转换），必要时显式转换并检查范围。
