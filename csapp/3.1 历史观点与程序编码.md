# 听课笔记：历史观点与程序编码

## 1. 这一章要解决什么问题？

你写了一段C代码，计算机是怎么执行它的？

```
你写的: int a = b + c;
计算机看到的: 一堆010101...
```

这一章就是要揭开这个"黑盒"：**从C代码到机器代码的翻译过程**。

理解这个有什么用？

- 排查莫名其妙的bug

- 理解程序是怎么"跑"起来的

## 2. 先说说历史（简单了解一下）

英特尔的处理器发展像"套娃"一样：

```
1978年: 8086 (16位) → 80386 (32位) → 现在的 x86-64 (64位)
        ↑                          ↑
        老祖宗                      能处理更多数据
```

**为什么要知道这个？** 因为很多术语（比如"字"）沿用了老定义，不学历史会觉得奇怪。

## 3. GCC编译：从C代码到可执行文件

### 3.1 整体流程图

```
源代码 (.c)
    ↓  编译
汇编代码 (.s)  ← 人能看懂的
    ↓  汇编
机器代码 (.o)  ← 电脑能执行的（二进制）
    ↓  链接
可执行文件
```

### 3.2 常用命令

| 你想做什么           | 命令                 | 得到什么 |
| -------------------- | -------------------- | -------- |
| 编译成可执行文件     | `gcc -o prog main.c` | prog     |
| 只生成汇编代码       | `gcc -S main.c`      | main.s   |
| 只生成机器代码       | `gcc -c main.c`      | main.o   |
| 查看二进制对应的汇编 | `objdump -d main.o`  | 汇编输出 |

### 3.3 关于优化级别 `-Og`

```bash
gcc -Og -o prog main.c
```

- `-Og`：轻度优化，汇编代码和C代码大致对应，**适合学习**
- `-O1`、`-O2`：更激进的优化，代码可能变得面目全非
- 学习阶段用 `-Og`，实际项目用 `-O2`

## 4. 寄存器：CPU的"快速记事本"

### 4.1 什么是寄存器？

想象你在做数学题：

- **内存** = 草稿纸（容量大，但拿取慢）
- **寄存器** = 脑子里记的数字（容量小，但立刻能用）

CPU做运算时，必须先把数据从内存"搬"到寄存器里。

### 4.2 16个通用寄存器

x86-64 有 16 个通用寄存器，每个 64 位（8字节）：

```
┌─────────────────────────────────────────┐
│  %rax  ←── 返回值（函数调用完的结果在这）    │
│  %rbx  ←── 通用（被调用者保存）            │
│  %rcx  ←── 第4个参数                      │
│  %rdx  ←── 第3个参数                      │
│  %rsi  ←── 第2个参数                      │
│  %rdi  ←── 第1个参数                      │
│  %rbp  ←── 通用（被调用者保存）            │
│  %rsp  ←── 栈指针（重要！指向栈顶）         │
│  %r8-%r15  ←── 额外的寄存器               │
└─────────────────────────────────────────┘
```

**记住这个口诀：**

- 返回值 → `%rax`
- 参数 → `%rdi, %rsi, %rdx, %rcx`（按顺序：第1、2、3、4个）

### 4.3 为什么有些寄存器叫"被调用者保存"？

**场景**：函数A调用函数B

```
函数A {
    x = 100;          // x 存在 %rbx 里
    调用函数B();       // B 可能会修改 %rbx！
    print(x);         // x 还是 100 吗？
}
```

两种解决策略：

| 策略         | 谁负责？      | 怎么做？                    | 哪些寄存器？                           |
| ------------ | ------------- | --------------------------- | -------------------------------------- |
| 调用者保存   | A（调用者）   | A在调用B前保存，B返回后恢复 | %rax, %rcx, %rdx, %rsi, %rdi, %r8-%r11 |
| 被调用者保存 | B（被调用者） | B开始时保存，返回前恢复     | %rbx, %rbp, %r12-%r15                  |

**简单理解**：

- 调用者保存寄存器 = "公共物品"，谁用谁负责
- 被调用者保存寄存器 = "私人财产"，用了要还原

## 5. 数据大小：为什么叫"字"？

### 5.1 历史遗留问题

8086是16位处理器，所以：

- **字(Word)** = 16位（2字节）← 这个命名固定下来了

后来扩展到32位、64位，但"字"还是指16位：

| C语言类型  | 大小  | 汇编后缀 | 名字 |
| ---------- | ----- | -------- | ---- |
| char       | 1字节 | b        | 字节 |
| short      | 2字节 | w        | 字   |
| int        | 4字节 | l        | 双字 |
| long, 指针 | 8字节 | q        | 四字 |

### 5.2 指令后缀的含义

```asm
movb  %al, %bl    ; 移动 1 字节
movw  %ax, %bx    ; 移动 2 字节
movl  %eax, %ebx  ; 移动 4 字节
movq  %rax, %rbx  ; 移动 8 字节
```

**后缀告诉你操作多少数据**，就像"买1个"和"买1箱"的区别。

## 6. 实战：读懂一段汇编代码

### 6.1 C代码

```c
void multstore(long x, long y, long *dest) {
    long t = mult2(x, y);   // 调用另一个函数
    *dest = t;              // 把结果存到 dest 指向的位置
}
```

### 6.2 对应的汇编代码

```asm
pushq   %rbx          ; ① 保存 %rbx（因为它是被调用者保存的）
movq    %rdx, %rbx    ; ② 把 dest 指针存到 %rbx
call    mult2         ; ③ 调用 mult2(x, y)
movq    %rax, (%rbx)  ; ④ 把返回值（在 %rax）存到 dest 指向的内存
popq    %rbx          ; ⑤ 恢复 %rbx
ret                   ; ⑥ 返回
```

### 6.3 逐行解释

| 步骤 | 指令                | 在干什么？                   | 为什么？                                                              |
| ---- | ------------------- | ---------------------------- | --------------------------------------------------------------------- |
| ①    | `pushq %rbx`        | 把 %rbx 的值压入栈保存       | %rbx 是被调用者保存的，用之前必须保存原值                             |
| ②    | `movq %rdx, %rbx`   | 把 %rdx 复制到 %rbx          | %rdx 是第3个参数(dest)，但后面调用 mult2 会覆盖 %rdx，所以先存到 %rbx |
| ③    | `call mult2`        | 调用 mult2                   | x 在 %rdi，y 在 %rsi，返回值会在 %rax                                 |
| ④    | `movq %rax, (%rbx)` | 把 %rax 存到 %rbx 指向的内存 | (%rbx) 表示"取 %rbx 里存的地址，往那个地址写数据"                     |
| ⑤    | `popq %rbx`         | 从栈中恢复 %rbx              | 还原 %rbx 的原值                                                      |
| ⑥    | `ret`               | 返回调用者                   | 函数结束                                                              |

### 6.4 关键概念：参数传递

调用 `mult2(x, y)` 时：

```
x → %rdi（第1个参数）
y → %rsi（第2个参数）
返回值 → %rax
```

这是 **调用约定**，所有人都要遵守。

---

## 7. 总结成一句话

- **编译**：C代码 → 汇编 → 机器码 → 可执行文件
- **寄存器**：CPU内部的高速存储，函数参数和返回值通过特定寄存器传递
- **保存策略**：被调用者保存的寄存器用完要还原，调用者保存的自己负责
- **指令后缀**：b/w/l/q 表示操作 1/2/4/8 字节

## 8. 延伸：如何自己验证？

```bash
# 写一个简单的C文件
echo 'int add(int a, int b) { return a + b; }' > add.c

# 生成汇编
gcc -S add.c

# 查看汇编代码
cat add.s
```

你会看到类似：

```asm
movl    %edi, %eax    ; 把第1个参数(a)放到 %eax
addl    %esi, %eax    ; 把第2个参数(b)加到 %eax
ret                    ; 返回（%eax 里就是结果）
```
