
# 听课笔记：整数在计算机系统中的表示

## 1. C语言中的整数数据类型
- **C语言支持多种整型**：如 `char`、`short`、`int` 等，用于指定不同大小的数据。
- **数据范围与字节数**：
  - 不同的数据类型占用的字节数不同，直接影响其数值范围。
  - **`long` 类型的大小**：
    - 与机器字长相关：
      - **64位机器**：`long` 占 8 字节。
      - **32位机器**：`long` 占 4 字节。
- **有符号与无符号**：
  - `unsigned`：限制数值为非负数（即**无符号数**）。
  - C语言同时支持**有符号数**（可正可负）和**无符号数**。

---

## 2. 无符号数的编码
- **表示方法**：
  - 假设数据类型有 `w` 位，用向量 `X` 表示，每个元素 `x_i` 取值为 0 或 1。
  - 映射函数：`B2U(X)`（Binary to Unsigned）。
  - 计算公式：
    ```
    B2U(X) = Σ (x_i * 2^i)
    ```
    类比十进制：`Σ (y_i * 10^i)`，区别在于权重基数为 2，且每位取值仅为 0 或 1。

### 示例
- `0101` → 计算：`1*4 + 0*2 + 1*1 = 5`
- `1011` → 计算：`1*8 + 0*4 + 1*2 + 1*1 = 11`

### 图形化表示（CSAPP方法）
- 每位 `i` 对应长度为 `2^i` 的条状图。
- 整个数的值为所有 `x_i=1` 的条状图长度之和。
- **取值范围**（4位为例）：
  - 最小值：`0000` → 0
  - 最大值：`1111` → 15
- **局限性**：只能表示非负数。

---

## 3. 有符号数的编码（补码）
- **表示方法**：
  - 最高位（符号位）的权重为负：`-2^(w-1)`。
  - 映射函数：`B2T(X)`（Binary to Two‘s complement）。
  - 计算公式：
    ```
    B2T(X) = -x_{w-1} * 2^{w-1} + Σ (x_i * 2^i)   (i 从 0 到 w-2)
    ```
  - 符号位为 1 时表示负数，为 0 时表示非负数。

### 示例
- `0101`：`-0*8 + 1*4 + 0*2 + 1*1 = 5`
- `1011`：`-1*8 + 0*4 + 1*2 + 1*1 = -5`

### 图形化表示
- 最高位为 1 时，灰色条状图表示负权重。
- **取值范围**（4位为例）：
  - 最小值：`1000` → -8
  - 最大值：`0111` → 7

---

## 4. 不同字长的数值范围
### 无符号数
- **8位**：最大值 = `2^8 - 1` = 255
- **16位**：最大值 = `2^16 - 1` = 65535
- 所有位为 1 时表示最大值。

### 有符号数（补码）
- **最大值**：
  - 8位：`2^7 - 1` = 127
  - 16位：`2^15 - 1` = 32767
  - 表示：符号位为 0，其余位为 1。
- **最小值**：
  - 8位：`-2^7` = -128（`1000 0000`）
  - 16位：`-2^15` = -32768（`1000 0000 0000 0000`）
- **特殊值 -1**：
  - 无论字长多少，-1 的补码表示为**全 1 串**。
  - 与无符号数的最大值具有相同的位模式。

> **注意**：C语言标准并未强制要求使用补码，但几乎所有现代机器都采用补码表示有符号数。

---

## 5. 实例分析
### 示例：12345 与 -12345
- **12345**（`short` 类型）：
  - 二进制位模式：`0011 0000 0011 1001`
  - 根据补码定义，权重相加可得 12345。
- **-12345**：
  - 位模式与 12345 不同（符号位为 1）。
  - 若将该位模式解释为无符号数：
    - 最高位的 1 不再表示负权重，而是正权重。
    - 计算结果为 53191。

### 结论
- **相同的位模式**，因映射函数不同，得到的数值不同。
- 无符号数与有符号数之间的转换将在后续视频中讲解。

---

## 重要概念与注意
- **补码表示**是当前计算机系统中事实上的标准。
- **-1 的表示**：全 1 串，易与无符号数的最大值混淆。
- **符号位的权重**：需理解为“负权重”，而不仅仅是“负号”。
- **取值范围**由字长和编码方式共同决定。
