# 听课笔记：数据传送与栈操作

## 1. 数据传送的系统视角

### 1.1 程序执行的数据流动

回顾第一章提到的 hello world 程序加载过程：

```
硬盘（可执行文件） → 内存（指令+数据） → CPU（执行）
```

程序执行过程中，CPU 和内存之间需要**频繁的数据存取**。

### 1.2 加法操作的数据传送示例

CPU 执行一个简单的加法操作 `a = a + b`：

1. **读取数据**：通过数据传送指令，将 A 和 B 的值从内存读到寄存器
2. **执行运算**：CPU 完成加法运算
3. **保存结果**：通过数据传送指令将计算结果保存到内存

### 1.3 寄存器与数据大小

以 64 位处理器为例，寄存器 `%rax` 的大小是 **64 位（8 字节）**：

| 变量类型 | 占用字节 | 使用的寄存器部分 | 寄存器符号 |
| -------- | -------- | ---------------- | ---------- |
| `long`   | 8 字节   | 全部 64 位       | `%rax`     |
| `int`    | 4 字节   | 低 32 位         | `%eax`     |
| `short`  | 2 字节   | 低 16 位         | `%ax`      |

> 虽然用了不同的表示符号，但实际上只是针对同一寄存器的不同数位进行操作。

### 1.4 exchange 函数示例

```c
long exchange(long *xp, long y) {
    long x = *xp;
    *xp = y;
    return x;
}
```

对应的汇编指令：

```asm
movq    (%rdi), %rax    ; 从内存读取 *xp 到 %rax（同时作为返回值）
movq    %rsi, (%rdi)    ; 将 y 写入 *xp 指向的内存
ret                      ; 返回
```

**寄存器使用惯例：**

| 寄存器 | 用途            |
| ------ | --------------- |
| `%rdi` | 第一个参数 `xp` |
| `%rsi` | 第二个参数 `y`  |
| `%rax` | 返回值 `x`      |

> **关键理解**：C 语言中的**指针其实就是地址**。

## 2. 栈操作指令

### 2.1 栈的基本概念

栈（Stack）是一种数据结构：

- **push**：把数据压入栈
- **pop**：删除栈顶数据
- **特性**：弹出的数据永远是最近被压入且仍在栈中的（LIFO）

### 2.2 程序栈的本质

程序栈是**内存的一个区域**：

- 栈的增长方向：**从高地址向低地址**
- 栈顶元素：所有栈中元素地址中**最低的**
- 栈顶指针：寄存器 `%rsp` 保存栈顶地址

> 惯例：栈是倒过来画的，栈顶在图中的底部，栈底在顶部。

### 2.3 pushq 指令

将数据压入栈，例如 `pushq %rax`：

**执行过程分解：**

1. 栈顶指针 `%rsp` 减 8（64 位数据）
2. 将数据复制到新的栈顶位置

**示例：**

```
压栈前：rsp = 0x108
压栈后：rsp = 0x100，内存 0x100 处保存 %rax 的值
```

**等效指令：**

```asm
pushq   %rax

; 等效于：
subq    $8, %rsp        ; 栈顶指针减 8
movq    %rax, (%rsp)    ; 数据写入新栈顶
```

> `pushq` 只需 1 个字节，而等效的两条指令需要 8 个字节。

### 2.4 popq 指令

从栈顶弹出数据，例如 `popq %rbx`：

**执行过程分解：**

1. 从栈顶位置读出数据复制到目标寄存器
2. 栈顶指针 `%rsp` 加 8

**示例：**

```
pop 前：rsp = 0x100，内存 0x100 处保存数据 0x123
pop 后：rsp = 0x108，%rbx = 0x123
```

**等效指令：**

```asm
popq    %rbx

; 等效于：
movq    (%rsp), %rbx    ; 从栈顶读取数据
addq    $8, %rsp        ; 栈顶指针加 8
```

> **注意**：pop 操作后，原栈顶内存位置的数据仍然存在，直到下次 push 操作才会被覆盖。
