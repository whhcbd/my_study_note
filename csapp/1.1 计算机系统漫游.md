## 2. 第一章核心：Hello World 程序的生命周期
本章通过一个最简单的打印程序 `hello.c`，概述计算机系统的编译与执行过程。

### 2.1 源代码编写
- **文件**：`hello.c`
- **工具**：文本编辑器 (如 vim, emacs 等)
- **示例代码**：
```c
#include <stdio.h>

int main() {
    printf("Hello World\n");
    return 0;
}
```

### 2.2 编译系统的四个阶段
在 Linux 环境下，通过一条命令（如 `gcc`）即可完成编译，但内部处理分为四个复杂阶段：

| 阶段 | 名称 | 输入文件 | 输出文件 | 主要工作 | 文件类型 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **1** | **预处理** (Preprocessing) | `hello.c` | `hello.i` | 处理以 `#` 开头的预处理指令：`#include` 插入头文件、`#define` 宏替换、`#ifdef` 条件编译等。 | 文本文件 |
| **2** | **编译** (Compilation) | `hello.i` | `hello.s` | 词法/语法/语义分析、中间代码生成及优化。翻译成汇编代码。 | 文本文件 |
| **3** | **汇编** (Assembly) | `hello.s` | `hello.o` | 将汇编代码翻译成机器指令，打包成**可重定位目标文件**。 | 二进制文件 (不可执行) |
| **4** | **链接** (Linking) | `hello.o` + 库文件 | `hello` | 合并目标文件 (如 `printf.o`)，解析符号，生成**可执行目标文件**。 | 二进制文件 (可执行) |

- **关于链接的补充**：
    - `printf` 函数位于 C 标准库（`libc.a` 静态库或 `libc.so` 动态库）中。
    - 链接器负责将 `hello.o` 与标准库合并。
    - 因为需要后续调整，所以 `hello.o` 被称为**可重定位目标文件**。
    - 最终生成的 `hello` 文件可被加载到内存中执行。

## 3. 为什么要理解编译系统？
虽然 IDE 可以一键编译，但理解底层原理对程序员至关重要：

### 3.1 优化程序性能
- **目的**：了解 C 代码如何转换为机器代码，从而写出更高效的代码。
- **常见问题**：
    - `switch` 语句是否比 `if-else` 高效？
    - 函数调用的开销有多大？
    - `while` 循环是否比 `for` 循环高效？
- **对应章节**：第 3 章、第 5 章。

### 3.2 理解链接错误
- **背景**：大型程序涉及多个函数库调用，链接错误常见且隐蔽。
- **常见问题**：
    - 静态变量与全局变量的区别？
    - 静态库与动态库的区别？
    - 运行时才出现的链接错误。
- **对应章节**：第 7 章。

### 3.3 避免安全漏洞
- **背景**：缓冲区溢出是互联网安全漏洞的主要原因。
- **核心**：理解数据和控制信息在程序栈上的存储方式，以及不规范书写的后果。
- **内容**：栈的原理、缓冲区溢出错误、利用编译器/OS 降低风险。
- **对应章节**：第 3 章。
