# 听课笔记：整数的运算（加法与减法）

## 1. 无符号数加法
### 示例
```c
unsigned char a = 255;
unsigned char b = 1;
unsigned char c = a + b; // 实际结果为 0，而不是 256
```
- **溢出**：结果超过了类型能表示的最大值（255），高位被丢弃。

### 无符号数加法原理
- 定义：对于 `w` 位的无符号数加法 `+_w^u`，操作数 `x, y` 满足 `0 ≤ x, y < 2^w`。
- **运算规则**：
  - 若 `x + y < 2^w`：结果 `s = x + y`。
  - 若 `x + y ≥ 2^w`：发生溢出，结果 `s = x + y - 2^w`（丢弃最高位）。
- **二进制解释**：和超过 `w` 位时，最高位的 1 被丢弃，相当于减去 `2^w`。

### 溢出检测
- 原理：无符号数加法不会减小数值，因此若 `s < x` 或 `s < y`，则说明发生了溢出。
```c
int uadd_ok(unsigned x, unsigned y) {
    unsigned s = x + y;
    return s >= x;  // 若 s ≥ x，则无溢出；否则溢出
}
```
- **数学推导**：
  - 无溢出时：`s = x + y ≥ x`。
  - 溢出时：`s = x + y - 2^w`，由于 `y < 2^w`，`y - 2^w < 0`，所以 `s = x + (y - 2^w) < x`。

## 2. 有符号数（补码）加法
### 定义
- 用 `+_w^t` 表示 `w` 位补码加法，操作数 `x, y` 满足 `-2^{w-1} ≤ x, y ≤ 2^{w-1}-1`。
- 真实和 `x + y` 的范围为 `[-2^w, 2^w - 2]`，需要 `w+1` 位表示。结果截断为 `w` 位。

### 运算规则
- 将 `x` 和 `y` 视为无符号数相加，然后对结果进行截断并解释为补码。等价于：
  - 若 `x + y ≥ 2^{w-1}`（正溢出）：结果 `s = x + y - 2^w`。
  - 若 `x + y < -2^{w-1}`（负溢出）：结果 `s = x + y + 2^w`。
  - 否则：结果 `s = x + y`。

### 示例
```c
char x = 64;   // 二进制 01000000
char y = 64;   // 二进制 01000000
char z = x + y; // 期望 128，实际 -128
```
- 计算：`x + y = 128`（二进制 `10000000`），解释为补码得 `-128`（正溢出）。
- **负溢出示例**：`-64 + (-64) = -128`（期望），但 `-128 + (-128)` 会负溢出为 0。

### 溢出检测
- **正溢出**：两个正数相加得到负数。
- **负溢出**：两个负数相加得到正数。
```c
int tadd_ok(int x, int y) {
    int s = x + y;
    if (x > 0 && y > 0 && s <= 0) return 0; // 正溢出
    if (x < 0 && y < 0 && s >= 0) return 0; // 负溢出
    return 1;
}
```

## 3. 减法与加法逆元（相反数）
### 加法逆元概念
- 对于给定 `x`，存在 `x'` 使得 `x + x' = 0`，称 `x'` 为 `x` 的加法逆元（即相反数）。
- 减法转换为加法：`y - x = y + (-x)`。

### 无符号数的逆元
- 由于无符号数非负，逆元通过溢出实现：`x + (2^w - x) = 2^w`，溢出后结果为 0。
- 公式：
  ```
  -_w^u x = 2^w - x   (x > 0)
  -_w^u 0 = 0
  ```
- 示例：对于 8 位无符号数，`x = 1`，则逆元为 `256 - 1 = 255`（即 `1 + 255 = 256` 溢出为 0）。

### 有符号数的逆元
- 对于补码，除 `TMin` 外，逆元为 `-x`（直接取负）。
- **特殊情况**：`TMin`（即 `-2^{w-1}`）的逆元是它本身，因为 `-TMin = 2^{w-1}` 超出范围，但通过负溢出 `TMin + TMin = -2^w` 溢出为 0。
- 公式：
  ```
  -_w^t x = 
      -x                (x > TMin)
      TMin              (x = TMin)
  ```
- 示例：8 位补码中，`TMin = -128`，`-128 + (-128) = -256`，截断为 8 位得 0，因此 `-128` 的逆元就是自身。

---

## 4. 关键点总结
- **无符号数加法**：溢出时丢弃进位，结果等于和模 `2^w`。检测溢出用 `s < x`。
- **补码加法**：分为正溢出、负溢出和正常。检测溢出看符号位变化。
- **减法**：通过加法逆元实现。无符号逆元为 `2^w - x`；补码逆元通常为 `-x`，但 `TMin` 的逆元是其本身。
- **C语言行为**：溢出不报错，结果按截断规则处理。程序员需自行检测溢出。

---

## 5. 注意事项
- 有符号数溢出是未定义行为（C标准），但大多数机器按补码规则处理。
- 编写安全代码时，应避免依赖溢出结果，或显式检测溢出。
- 理解加法逆元有助于分析减法运算的底层实现。
