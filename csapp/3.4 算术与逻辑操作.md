# 听课笔记：算术与逻辑操作

## 1. 加载有效地址指令 LEAQ

### 1.1 指令功能

**LEAQ**（Load Effective Address，加载有效地址）：
- Q 表示地址长度为 4 字（64 位）
- 64 位处理器上地址长度都是 64 位，因此不存在 LEAB、LEAW 等变种

### 1.2 有效地址计算

源操作数格式与**内存引用**类似，但执行的操作不同：

```asm
leaq    7(%rdx, %rdx, 4), %rax
```

假设 `%rdx` 保存的值为 x，则：
- **有效地址** = 7 + x + 4×x = **5x + 7**
- **注意**：LEAQ **不访问内存**，而是直接将计算出的地址值写入目的寄存器

> 与内存引用的区别：内存引用会去该地址读取数据，而 LEAQ 只计算地址值本身。

### 1.3 LEAQ 用于算术运算

LEAQ 可用于实现**加法和有限的乘法运算**。

**C 代码示例：**

```c
long scale(long x, long y, long z) {
    long t = x + 4 * y + 12 * z;
    return t;
}
```

**编译后的汇编代码：**

```asm
leaq    (%rdi, %rsi, 4), %rax      ; x + 4*y → %rax
leaq    (%rdx, %rdx, 2), %rdx      ; z + 2*z = 3*z → %rdx
leaq    (%rax, %rdx, 4), %rax      ; (x+4*y) + 4*(3*z) = x+4*y+12*z → %rax
```

**寄存器使用惯例：**

| 寄存器 | 参数 |
| ------ | ---- |
| `%rdi` | x    |
| `%rsi` | y    |
| `%rdx` | z    |

### 1.4 乘法分解的原因

**为什么不直接一步计算 z×12？**

因为**比例因子**只能是 1、2、4、8 中的一个，所以需要分解：
- 第一步：`3×z`（比例因子 2，即 z + 2z）
- 第二步：`4×(3z)`（比例因子 4，即左移 2 位）

## 2. 一元操作指令

这类指令只有**一个操作数**，该操作数既是源操作数也是目的操作数。

| 指令 | 含义 |
| ---- | ---- |
| `INC` | 加 1 |
| `DEC` | 减 1 |
| `NEG` | 取负 |
| `NOT` | 按位取反 |

**操作数可以是：**
- 寄存器
- 内存地址

---

## 3. 二元操作指令

这类指令有**两个操作数**：

| 操作数   | 说明                                         |
| -------- | -------------------------------------------- |
| 第一个   | 源操作数：立即数、寄存器或内存地址           |
| 第二个   | 源操作数 + 目的操作数：寄存器或内存地址      |

> 第二个操作数**不能是立即数**。

**常用指令：**

| 指令  | 含义     |
| ----- | -------- |
| `ADD` | 加法     |
| `SUB` | 减法     |
| `IMUL`| 乘法     |
| `XOR` | 异或     |
| `OR`  | 或       |
| `AND` | 与       |

---

## 4. 指令执行示例

**初始状态：**

| 位置           | 值       |
| -------------- | -------- |
| 内存 0x100     | 0xFF     |
| 内存 0x108     | 0xAB     |
| 内存 0x110     | 0x13     |
| 寄存器 %rax    | 0x100    |
| 寄存器 %rcx    | 0x1      |
| 寄存器 %rdx    | 0x3      |

**指令执行过程：**

```asm
addq    %rcx, 0x100     ; 内存 0x100: 0xFF + 0x1 = 0x100
subq    %rdx, 0x108     ; 内存 0x108: 0xAB - 0x3 = 0xA8
incq    0x110           ; 内存 0x110: 0x13 + 1 = 0x14
subq    %rdx, %rax      ; 寄存器 %rax: 0x100 - 0x3 = 0xFD
```

## 5. 移位操作指令

### 5.1 指令分类

| 指令        | 含义         | 填充方式     |
| ----------- | ------------ | ------------ |
| `SAL`/`SHL` | 左移         | 右边填 0     |
| `SAR`       | 算术右移     | 填符号位     |
| `SHR`       | 逻辑右移     | 填 0         |

> 左移指令 SAL 和 SHL 效果相同。

### 5.2 移位量的指定

移位量 K 可以是：
- **立即数**
- **寄存器 %cl**（只能用这个特定的寄存器）

### 5.3 移位量的有效位数

寄存器 CL 长度为 8 位，但实际移位量由 CL 的**低 m 位**决定：

| 操作数大小 | m 值 | 有效位数    |
| ---------- | ---- | ----------- |
| 字节 (B)   | 3    | CL[0:2]     |
| 字 (W)     | 4    | CL[0:3]     |
| 双字 (L)   | 5    | CL[0:4]     |
| 四字 (Q)   | 6    | CL[0:5]     |

---

## 6. 移位指令应用示例

**C 代码：**

```c
long scale2(long x, long y, long z) {
    long t = x + 4 * y + 48 * z;
    return t;
}
```

**编译后的汇编代码：**

```asm
leaq    (%rdi, %rsi, 4), %rax      ; x + 4*y → %rax
leaq    (%rdx, %rdx, 2), %rdx      ; 3*z → %rdx
salq    $4, %rdx                   ; 3*z * 16 = 48*z
addq    %rdx, %rax                 ; (x+4*y) + 48*z
```

**分解 48 的方式：**
- `48 = 3 × 16`
- `3 = 1 + 2`（通过 LEAQ 实现）
- `×16` 通过左移 4 位实现（`salq $4`）

### 6.1 为什么不用乘法指令？

- **乘法指令执行时间更长**
- 编译器会优先选择更高效的方式（LEAQ + 移位）
