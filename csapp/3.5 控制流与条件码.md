# 听课笔记：控制流与条件码

## 1. 概述

在C语言中，**条件语句**和**循环语句**等控制流语句需要根据条件判断结果来决定操作执行的顺序。

**算术逻辑单元（ALU）**：
- 执行算术和逻辑运算指令
- 从寄存器读取数据，执行运算，将结果返回到目的寄存器
- 根据运算结果设置**条件码寄存器**

---

## 2. 条件码寄存器

### 2.1 基本概念

**条件码寄存器**：
- 由 CPU 维护
- 长度为**单个比特位**
- 描述**最近执行操作**的属性
- 会被后续指令的执行结果覆盖

**示例**：T1 时刻执行指令1，T2 时刻执行指令2 → 条件码寄存器的内容会被指令2的结果覆盖。

### 2.2 常用条件码

| 标志 | 名称 | 含义 | 用途 |
| ---- | ---- | ---- | ---- |
| **CF** | 进位标志 (Carry Flag) | 最高位产生进位 | 检查**无符号数**溢出 |
| **ZF** | 零标志 (Zero Flag) | 操作结果等于 0 | 判断结果是否为零 |
| **SF** | 符号标志 (Sign Flag) | 操作结果小于 0（最高位为1） | 判断结果正负 |
| **OF** | 溢出标志 (Overflow Flag) | 有符号数发生正溢出或负溢出 | 检查**有符号数**溢出 |

### 2.3 条件码示例

**CF 进位标志示例**（无符号数溢出）：
```
A = 255, B = 1
A + B = 256 → 最高位产生进位 → CF = 1
```

**ZF 零标志示例**：
```
A + B = 0 → ZF = 1
```

---

## 3. 设置条件码的指令

### 3.1 算术和逻辑运算指令

这些指令会改变条件码寄存器：

| 指令类型 | 示例 | 条件码设置规则 |
| -------- | ---- | -------------- |
| 逻辑操作 | XOR | CF 和 OF 置 0 |
| 加一指令 | INC | 设置 OF 和 ZF，**不改变 CF** |
| 减一指令 | DEC | 设置 OF 和 ZF，**不改变 CF** |

### 3.2 CMP 指令

**CMP**（Compare）指令：
- 根据**两个操作数的差**来设置条件码
- 与**减法指令**类似，但**不更新目的寄存器**的值

```asm
cmp    %rsi, %rdi    ; 根据 %rdi - %rsi 的结果设置条件码，%rdi 值不变
```

### 3.3 TEST 指令

**TEST** 指令：
- 与 **AND** 指令类似
- 只设置条件码寄存器，**不改变目的寄存器**的值

```asm
test   %rax, %rax    ; 检查 %rax 是否为 0
```

## 4. 条件码的使用

### 4.1 SET 类指令

**通常不会直接读取条件码寄存器**，而是使用 **SET 类指令**：
- 根据条件码的某种组合
- 将一个**字节**设置为 0 或 1

### 4.2 示例：判断 A == B

**C 代码：**
```c
int equal(long a, long b) {
    return a == b;
}
```

**汇编代码：**
```asm
cmp    %rsi, %rdi    ; a - b，设置条件码
sete   %al           ; 若 ZF = 1（相等），则 %al = 1；否则 %al = 0
movzbl %al, %eax     ; 零扩展，返回结果
```

**解析**：
- 参数 a 在 `%rdi`，参数 b 在 `%rsi`
- `cmp` 指令：当 a == b 时，a - b = 0，ZF = 1
- `sete`（Set Equal）：后缀 **E** 是 **Equal** 的缩写，根据 ZF 设置 `%al`

### 4.3 示例：判断 A < B（有符号数）

**C 代码：**
```c
int less(long a, long b) {
    return a < b;
}
```

**汇编代码：**
```asm
cmp    %rsi, %rdi    ; a - b，设置条件码
setl   %al           ; 若 a < b，则 %al = 1；否则 %al = 0
movzbl %al, %eax     ; 零扩展，返回结果
```

**解析**：
- `setl`（Set Less）：后缀 **L** 是 **Less** 的缩写
- 判断 a < b 需要 **SF（符号标志）** 和 **OF（溢出标志）** 的**异或结果**

---

## 5. 有符号数比较的条件码组合

### 5.1 为什么需要 SF ^ OF？

**情况分析**（两个有符号数相减：a - b）：

| 情况 | a 与 b 的关系 | 结果 | SF | OF | SF ^ OF |
| ---- | ------------- | ---- | -- | -- | ------- |
| 无溢出，结果为负 | a < b | 负数 | 1 | 0 | **1**（真） |
| 无溢出，结果为正 | a > b | 正数 | 0 | 0 | **0**（假） |
| 负溢出 | a = -2, b = 127 | 127（正） | 0 | 1 | **1**（真） |
| 正溢出 | a = 1, b = -128 | -127（负） | 1 | 1 | **0**（假） |

### 5.2 溢出情况详解

**负溢出示例**：
```
a = -2 (1111...1110)
b = 127 (0111...1111)
a - b = -129，但 8 位表示结果为 127（正数）
SF = 0, OF = 1 → SF ^ OF = 1（a < b 为真）
```

**正溢出示例**：
```
a = 1 (0000...0001)
b = -128 (1000...0000)
a - b = 129，但 8 位表示结果为 -127（负数）
SF = 1, OF = 1 → SF ^ OF = 0（a < b 为假，实际 a > b）
```

**结论**：`a < b` 的判断条件是 **SF ^ OF = 1**

---

## 6. 条件码组合汇总

### 6.1 有符号数比较（SET 指令后缀）

| 后缀 | 含义 | 条件 |
| ---- | ---- | ---- |
| **E** / **NE** | Equal / Not Equal | ZF / ~ZF |
| **L** / **LE** | Less / Less or Equal | (SF ^ OF) / (SF ^ OF) \| ZF |
| **G** / **GE** | Greater / Greater or Equal | ~(SF ^ OF) & ~ZF / ~(SF ^ OF) |

### 6.2 无符号数比较

对于**无符号数**比较，使用 **CF（进位标志）** 和 **ZF（零标志）** 的组合：

| 后缀 | 含义 | 条件 |
| ---- | ---- | ---- |
| **E** / **NE** | Equal / Not Equal | ZF / ~ZF |
| **B** / **BE** | Below / Below or Equal | CF / CF \| ZF |
| **A** / **AE** | Above / Above or Equal | ~CF & ~ZF / ~CF |

> **注意**：无符号数比较使用 **B (Below)** 和 **A (Above)**，而不是 L 和 G。

---

## 7. 总结

1. **条件码寄存器**由 ALU 维护，描述最近操作的属性
2. **常用条件码**：CF（进位）、ZF（零）、SF（符号）、OF（溢出）
3. **设置条件码的指令**：
   - 算术和逻辑运算指令
   - CMP 指令（比较，不修改寄存器）
   - TEST 指令（测试，不修改寄存器）
4. **SET 类指令**：根据条件码组合将字节设为 0 或 1
5. **有符号数比较**：使用 SF ^ OF 判断大小
6. **无符号数比较**：使用 CF 和 ZF 的组合

> 了解条件语句的底层实现有助于深入理解计算机系统。
