# 📚 听课笔记：结构体内存布局、数据对齐与联合体

## 📌 核心概述

本期内容深入剖析 C 语言结构体的底层内存模型、**数据对齐（Data Alignment）** 机制及其对内存占用的影响，并讲解联合体（Union）的共享内存特性、标签联合设计模式及位模式访问（Type Punning）原理。

---

## 🔹 一、 结构体内存布局与字段访问

### 1. 内存分布特征

- 字段按声明顺序连续存放（受对齐规则影响会产生间隙）。
- **数组嵌入**：结构体内的数组元素直接嵌入结构体内存块中，不额外分配独立空间。
- **访问本质**：所有字段/数组元素均通过 `基地址 + 字节偏移量` 定位。

### 2. 汇编级访问逻辑

- 结构体指针存于寄存器（如 `%rdi`）
- 字段访问：`mov offset(%rdi), %reg`（偏移量由编译器静态计算）
- 数组元素地址：`基址(%rdi) + 索引 × 元素大小`，计算结果存入地址寄存器后解引用。
- 💡 **结论**：结构体字段访问与一维数组寻址在汇编层面**原理完全一致**。

---

## 🔹 二、 数据对齐原则（🔥核心机制）

### 1. 为什么需要对齐？

- **硬件访存效率**：CPU 内存总线通常按固定宽度（如 4B/8B）读取。未对齐访问可能触发多次内存周期或硬件异常，严重降低性能。

### 2. 对齐硬性规则

📏 **任何 K 字节的基本数据类型，其起始地址必须是 K 的倍数。**

| 数据类型 | 大小(K) | 对齐要求(地址必须是K的倍数) |
|---|---|---|
| `char` | 1 | 任意地址 |
| `short` | 2 | 2的倍数 |
| `int` / `float` / 32位指针 | 4 | 4的倍数 |
| `double` / `long` / 64位指针 | 8 | 8的倍数 |

### 3. 编译器的填充策略

- **字段间填充**：若当前偏移不满足下一字段对齐要求，自动插入 `Padding` 字节。
- **末端填充**：结构体总大小必须是其**内部最大对齐要求的倍数**。  
  _目的：保证结构体数组中，每个元素的起始地址都满足对齐规则。_

## 🔹 三、 复杂结构体大小计算示例

以讲座中的复杂结构体为例（**假设运行在64位系统，指针大小为8字节**）：

| 字段          | 类型        | 大小        | 对齐要求   | 起始偏移 | 说明                 |
| ----------- | --------- | --------- | ------ | ---- | ------------------ |
| `a`         | 指针        | 8B        | 8      | `0`  | 起始                 |
| `b`         | `short`   | 2B        | 2      | `8`  | 满足                 |
| _(Padding)_ | -         | **6B**    | -      | -    | 使 `c` 对齐到8的倍数      |
| `c`         | `double`  | 8B        | 8      | `16` | 满足                 |
| `d`         | `char`    | 1B        | 1      | `24` | 满足                 |
| _(Padding)_ | -         | **3B**    | -      | -    | 使 `e` 对齐到4的倍数      |
| `e`         | `int`     | 4B        | 4      | `28` | 满足                 |
| `f`         | `char`    | 1B        | 1      | `32` | 满足                 |
| _(Padding)_ | -         | **7B**    | -      | -    | 使 `g` 对齐到8的倍数      |
| `g`         | `long`/指针 | 8B        | 8      | `40` | 满足                 |
| `h`         | `int`     | 4B        | 4      | `48` | 满足                 |
| **末端填充**    | -         | **4B**    | -      | -    | 凑齐至 `56` (56%8==0) |
| **总计**      | -         | **56 字节** | 最大对齐=8 |      |                    |

> ✅ **优化技巧**：合理调整字段声明顺序（如按大小降序/升序排列）可显著减少 Padding 空间浪费。

## 🔹 四、 联合体（Union）机制与应用

### 1. 核心特性

- **内存共享**：所有字段重叠在同一块存储区域。
- **大小计算**：`sizeof(union) = max(sizeof(各字段))`，并满足最大字段的对齐要求。
- **写入覆盖**：修改任意字段会直接影响其他字段的位表示。

### 2. 典型应用场景

#### 🟢 场景一：互斥状态节省空间（标签联合 Tagged Union）

- **需求**：二叉树节点要么是内部节点（含2个子指针），要么是叶子节点（含2个double值），二者互斥。
- **方案对比**：
  - 纯结构体：`32B`（始终分配所有字段空间）
  - 联合体：`16B`（`max(16B指针, 16B双精度)`）
  - **带标签结构体**：`4B(enum type) + 4B(padding) + 16B(union) = 24B`
- **权衡**：字段较少时节省有限；字段庞大或变体较多时，空间收益显著。

#### 🔵 场景二：位模式访问（Type Punning）

- **原理**：联合不同数据类型（如 `double` 与 `unsigned long`），以类型 A 写入，以类型 B 读取。
- **作用**：直接观察/操作底层二进制位，避免强制类型转换带来的**值语义截断或浮点转换**。
- ⚠️ **注意**：现代 C 标准（C11+）的**严格别名规则（Strict Aliasing）** 可能对此行为未定义，但在系统编程、编译器实现或嵌入式开发中仍广泛使用（常用 `memcpy` 替代以规避风险）。

## 📝 核心考点 & 面试高频题

1. **对齐三定律**：
   - 单个变量地址 `% K == 0`
   - 结构体总大小必须是最大对齐要求的倍数
   - 数组元素间距 = 结构体总大小（保证数组内每个元素对齐）
2. **结构体内存优化**：按字段大小从大到小声明，或按对齐要求分组声明。
3. `struct` vs `union`：结构体是**独立内存累加**，联合体是**共享内存取最大**。
4. **类型双关的本质**：同一块物理内存，不同逻辑类型的解释视图。
5. **偏移量计算**：笔试常考手动推算结构体各字段偏移及 `sizeof` 结果，务必注意**末端填充**。

| `V模式` | 位模式 (Bit Pattern) | 二进制底层表示 |

> 💡 **延伸思考**：为什么64位系统下 `struct { char c; int i; }` 的大小是 8 而不是 5？  
> _提示：`c` 占1B → 偏移0；`i` 需4B对齐 → 偏移4（中间3B Padding）；结构体最大对齐=4，总大小需为4的倍数 → 末端无需填充。实际总大小 = 1 + 3(pad) + 4 = 8B。_
