## 一、信息的存储：内存模型 🧠

### 1.1 内存的本质
```markdown
✅ 视图：程序将内存视为一个巨大的【字节数组】
✅ 寻址：每个字节都有唯一的数字编号 → 【地址】
✅ 集合：所有地址的集合 → 【虚拟地址空间】

📦 基本单元：字节 (Byte)
   • 1 字节 = 8 位 (bits)
   • 每位状态：0 或 1
   • 取值范围：00000000 (0) ~ 11111111 (255)
```

### 1.2 为什么需要十六进制？
| 进制 | 特点 | 缺点 |
|------|------|------|
| **二进制** | 机器原生，精确 | 太冗长 (11110000...) |
| **十进制** | 人类习惯 | 与二进制转换麻烦 |
| **十六进制** | **简洁 + 转换方便** | 需记忆 A-F |

```markdown
🔑 十六进制规则：
• 数字：0-9, 字母：A-F (代表 10-15)
• C 语言表示：0x 或 0X 开头 (如 0x1A3F)
• 大小写不敏感：0x1a3f == 0x1A3F
```

## 二、进制转换技巧 🔄

### 2.1 二进制 ↔ 十六进制（核心技巧）
```markdown
✅ 核心关系：1 个十六进制位 = 4 个二进制位

🔄 十六进制 → 二进制：
   直接展开每位即可
   例：0xA (1010) 0xF (1111) → 10101111

🔄 二进制 → 十六进制：
   1️⃣ 从右向左，每 4 位一组
   2️⃣ 最左边不足 4 位 → 补 0
   3️⃣ 每组转换为对应的十六进制数
   
   例：1011011011
   → 0010 | 1101 | 1011  (补 0 分组)
   →   2  |   D  |   B
   → 0x2DB
```

### 2.2 2 的幂次 → 十六进制（快速法）
```markdown
🎯 问题：2^N 如何快速转十六进制？

🧮 公式：N = 4q + r  (q 是商，r 是余数 0~3)
   • 十六进制结果 = 2^r 后面跟 q 个 0

📝 示例：2^11
   11 = 4×2 + 3  (q=2, r=3)
   • 2^3 = 8
   • 后面跟 2 个 0
   → 0x800

📝 示例：2^20
   20 = 4×5 + 0  (q=5, r=0)
   • 2^0 = 1
   • 后面跟 5 个 0
   → 0x100000
```

### 2.3 十进制 ↔ 十六进制
```markdown
🔹 十进制 → 十六进制：【辗转相除法】
   不断除以 16，取余数，直到商为 0
   余数逆序排列即为结果
   
   例：314156
   314156 ÷ 16 = 19634 ... 余 12 (C)
   19634  ÷ 16 = 1227  ... 余 2  (2)
   1227   ÷ 16 = 76    ... 余 11 (B)
   76     ÷ 16 = 4     ... 余 12 (C)
   4      ÷ 16 = 0     ... 余 4  (4)
   → 结果：0x4CB2C

🔹 十六进制 → 十进制：【幂次加权】
   每位数值 × 16^位置权值，然后求和
   例：0x7AF
   = 7×16² + 10×16¹ + 15×16⁰
```

> 💡 **工具建议**：大数值转换建议使用计算器或编程工具，手动转换易出错。

## 三、字长与虚拟地址空间 📏

### 3.1 字长 (Word Size) 的决定性作用
```markdown
🔍 定义：计算机一次能处理的天然整数大小（通常等于指针大小）
📐 公式：w 位机器 → 虚拟地址范围 0 ~ 2^w - 1
```

### 3.2 32 位 vs 64 位
| 特性 | 32 位机器 | 64 位机器 |
|------|----------|----------|
| **字长 w** | 32 bits | 64 bits |
| **最大寻址空间** | 2^32 ≈ 4 GB | 2^64 ≈ 16 EB (巨大) |
| **指针大小** | 4 字节 | 8 字节 |
| **现状** | 嵌入式常见 | 主流服务器/PC/手机 |

### 3.3 兼容性说明 ⚠️
```markdown
✅ 64 位机器兼容 32 位程序：
   • 大多数 64 位系统做了向后兼容
   • 命令：gcc -m32 hello.c  (生成 32 位可执行文件)

❌ 32 位机器无法运行 64 位程序：
   • 硬件不支持 64 位地址和指令

📌 关键区别：
   • 区别在于【程序是如何编译的】，而不是运行机器的类别
   • hello32 → 可跑在 32 位 & 64 位机器
   • hello64 → 只能跑在 64 位机器
```

## 四、C 语言数据类型的大小 📊

### 4.1 常见数据类型字节大小表
> ⚠️ **注意**：`int` 的大小可能随机器变化，但 `long` 和 `指针` 在 32/64 位下区别明显。

| 数据类型 | 32 位机器 (字节) | 64 位机器 (字节) | 说明 |
| :--- | :---: | :---: | :--- |
| `char` | 1 | 1 | 始终为 1 |
| `short` | 2 | 2 | |
| `int` | 4 | 4 | 通常不变 |
| `long` | 4 | 8 | **注意变化** |
| `long long` | 8 | 8 | |
| `float` | 4 | 4 | 单精度浮点 |
| `double` | 8 | 8 | 双精度浮点 |
| `void *` (指针) | 4 | 8 | **随字长变化** |

### 4.2 多字节数据的存储地址
```markdown
📍 场景：int x = 0x01234567; 地址从 0x100 开始
• int 占 4 字节 → 占用 0x100, 0x101, 0x102, 0x103

❓ 问题：哪个字节放在 0x100？
   • 最高有效字节 (0x01)？
   • 还是最低有效字节 (0x67)？
   → 这取决于【字节序】(Endianness)
```

## 五、字节序：大端法 vs 小端法 🔄

### 5.1 核心定义
```markdown
🔹 大端法 (Big-Endian)：
   • 【最高有效字节】存储在【最小地址】
   • 符合人类阅读习惯 (从左到右)
   • 代表：IBM, Sun (大多数), 网络协议

🔹 小端法 (Little-Endian)：
   • 【最低有效字节】存储在【最小地址】
   • 硬件实现更方便 (地址对齐)
   • 代表：Intel, AMD (大多数), Android, iOS
```

### 5.2 图解对比 (以 0x01234567 为例)

```
地址      大端法 (Big)    小端法 (Little)
----      ------------    ---------------
0x100     01 (最高)       67 (最低)
0x101     23              45
0x102     45              23
0x103     67 (最低)       01 (最高)

💡 记忆技巧：
   小端 = "小"地址存"低"位 (Little End at Low Address)
```

### 5.3 特殊情况
```markdown
🔸 双端架构：
   • 如 ARM 处理器，硬件支持配置成大端或小端
   • 但操作系统通常限定模式 (如 Android/iOS 强制小端)

🔸 实验观察：
   • 在不同机器上运行字节打印程序
   • 数值 12345 (0x3039) 的字节顺序会不同
   • 指针地址值在不同系统上完全不同 (32 位 vs 64 位)
```

## 六、有趣的现象：Int vs Float 🔍

### 6.1 实验观察
```markdown
🔹 对象：数值 12345
🔹 类型 1：int (整型)
   • 十六进制：0x00003039
   • 字节模式：相对简单

🔹 类型 2：float (浮点型)
   • 十六进制：0x4640E400 (示例值)
   • 字节模式：完全不同！

🔹 二进制位匹配：
   • 将两者转为二进制并移位
   • 发现存在长达 13 位的匹配序列！
   • 这不是巧合 → 浮点数编码有特定规则 (IEEE 754)
```
