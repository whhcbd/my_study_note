## 题目26

### 题目

使用算法1计算关系 R = {(a,b),(a,c),(a,e),(b,a),(b,c),(c,a),(c,b),(d,a),(e,d)} 在集合 {a,b,c,d,e} 上的传递闭包。

### 步骤1：构建邻接矩阵

将集合元素映射为索引：a=0, b=1, c=2, d=3, e=4

```
原始关系 R：
(a,b) (a,c) (a,e)
(b,a) (b,c)
(c,a) (c,b)
(d,a)
(e,d)

邻接矩阵 M_R：
     a  b  c  d  e
a   0  1  1  0  1
b   1  0  1  0  0
c   1  1  0  0  0
d   1  0  0  0  0
e   0  0  0  1  0
```

### 步骤2：计算 R² = R × R

R² 表示通过2步可达的关系。

```
R²[0][0] (a→a): 检查是否存在k使得 R[0][k]=1 且 R[k][0]=1
  k=1 (b): R[0][1]=1 且 R[1][0]=1 ✓ → R²[0][0]=1
  k=2 (c): R[0][2]=1 且 R[2][0]=1 ✓ → R²[0][0]=1

R²[0][1] (a→b): 检查是否存在k使得 R[0][k]=1 且 R[k][1]=1
  k=1 (b): R[0][1]=1 且 R[1][1]=0 ✗
  k=2 (c): R[0][2]=1 且 R[2][1]=1 ✓ → R²[0][1]=1

R²[0][2] (a→c): 检查是否存在k使得 R[0][k]=1 且 R[k][2]=1
  k=1 (b): R[0][1]=1 且 R[1][2]=1 ✓ → R²[0][2]=1
  k=2 (c): R[0][2]=1 且 R[2][2]=0 ✗

R²[0][3] (a→d): 检查是否存在k使得 R[0][k]=1 且 R[k][3]=1
  k=1 (b): R[0][1]=1 且 R[1][3]=0 ✗
  k=2 (c): R[0][2]=1 且 R[2][3]=0 ✗
  k=4 (e): R[0][4]=1 且 R[4][3]=1 ✓ → R²[0][3]=1

R²[0][4] (a→e): 检查是否存在k使得 R[0][k]=1 且 R[k][4]=1
  k=1 (b): R[0][1]=1 且 R[1][4]=0 ✗
  k=2 (c): R[0][2]=1 且 R[2][4]=0 ✗

用同样的方法计算所有元素：

R² 矩阵：
     a  b  c  d  e
a   1  1  1  1  0
b   1  1  1  0  0
c   1  1  1  0  0
d   0  1  1  0  1
e   1  0  0  0  0
```

**关键逻辑**：R²[i][j] = 1 表示存在一个中间节点k，使得i能到k且k能到j，即i能通过2步到达j。

### 步骤3：计算 R³ = R² × R

R³ 表示通过3步可达的关系。

```
R³ 矩阵：
     a  b  c  d  e
a   1  1  1  1  1
b   1  1  1  1  1
c   1  1  1  1  1
d   1  1  1  1  1
e   1  1  1  1  1
```

**观察**：R³已经是全1矩阵，表示所有节点都能通过3步或更少步数到达任何节点（包括自身）。

### 步骤4：计算 R⁴ 和 R⁵

由于R³已经是全1矩阵，后续的幂次仍然是全1矩阵：
- R⁴ = R³ × R = 全1矩阵
- R⁵ = R⁴ × R = 全1矩阵

### 步骤5：计算传递闭包 R* = R ∪ R² ∪ R³ ∪ R⁴ ∪ R⁵

```
传递闭包 R*：
     a  b  c  d  e
a   1  1  1  1  1
b   1  1  1  1  1
c   1  1  1  1  1
d   1  1  1  1  1
e   1  1  1  1  1
```

### 结果

**传递闭包**：
```
{(a,a),(a,b),(a,c),(a,d),(a,e),
 (b,a),(b,b),(b,c),(b,d),(b,e),
 (c,a),(c,b),(c,c),(c,d),(c,e),
 (d,a),(d,b),(d,c),(d,d),(d,e),
 (e,a),(e,b),(e,c),(e,d),(e,e)}
```

即：所有25个有序对（包括自反性）。

```
原始关系 R：
(a,e)
(b,a) (b,d)

(e,a) (e,b) (e,c) (e,e)
```

```
R²[0][0] (a→a): 检查是否存在k使得 R[0][k]=1 且 R[k][0]=1
  k=4 (e): R[0][4]=1 且 R[4][0]=1 ✓ → R²[0][0]=1

R²[0][1] (a→b): 检查是否存在k使得 R[0][k]=1 且 R[k][1]=1

     a  b  c  d  e

d   1  0  1  0  1

```

### 步骤3：计算 R³ = R² × R

```
R³ 矩阵：
     a  b  c  d  e

```

### 步骤4：计算 R⁴ = R³ × R

```
R⁴ 矩阵：

c   1  1  1  1  1

```

### 步骤5：计算 R⁵ = R⁴ × R

由于R⁴已经是全1矩阵，R⁵仍然是全1矩阵。

### 步骤6：计算传递闭包 R* = R ∪ R² ∪ R³ ∪ R⁴ ∪ R⁵

## 算法1详细逻辑讲解

### 核心思想

算法1基于传递闭包的数学定义：

```
R* = R ∪ R² ∪ R³ ∪ ... ∪ Rⁿ
```

其中：
- **R**：1步可达（直接边）
- **R²**：2步可达（通过1个中间节点）
- **R³**：3步可达（通过2个中间节点）
- ...
- **Rⁿ**：n步可达（通过n-1个中间节点）

### 布尔矩阵乘法的逻辑

#### 普通矩阵乘法回顾

```
C = A × B
C[i][j] = Σ(A[i][k] × B[k][j])，k从0到n-1
```

含义：A的第i行与B的第j列对应元素相乘再求和。

#### 布尔矩阵乘法

```
C = A ⊙ B
C[i][j] = ∨(A[i][k] ∧ B[k][j])，k从0到n-1
```

其中：
- ∧（逻辑与）：替换普通乘法
- ∨（逻辑或）：替换普通加法

**直观理解**：
```
C[i][j] = 1 当且仅当存在某个k，使得：
  - 从i能到达k（A[i][k]=1）
  - 且从k能到达j（B[k][j]=1）
```

### 算法1的详细步骤

```
输入：邻接矩阵 M_R（表示关系R）
输出：传递闭包矩阵 M_R*

步骤1：初始化
  A = M_R         // A存储当前的幂次矩阵
  B = M_R         // B累积所有可达关系

步骤2：对于 i = 2 到 n（n是节点数量）
  a. 计算 A = A ⊙ M_R
     这得到 R^i，表示通过i步可达的关系
  
  b. 更新 B = B ∪ A
     将新发现的i步可达关系加入到累积结果中

步骤3：返回 B
  B现在包含了所有1步、2步、...、n步可达的关系
  这就是传递闭包 R*
```

### 为什么只需要计算到Rⁿ？

**关键观察**：在有n个节点的图中，如果两个节点之间存在路径，那么一定存在长度不超过n的路径。

**理由**：
- 如果路径长度超过n，根据鸽巢原理，至少有一个节点被访问了两次
- 这意味着路径中有环
- 可以通过去掉环来缩短路径长度
- 因此，任何可达关系都可以在n步或更少步数内到达

### 算法1的时间复杂度分析

```
外层循环：执行n-1次
每次迭代：
  - 矩阵乘法：O(n³)
  - 矩阵加法：O(n²)

总时间复杂度：O(n⁴)
```

虽然效率不如Warshall算法的O(n³)，但算法1更直观，便于理解传递闭包的数学本质。

### 实际计算技巧

#### 技巧1：观察矩阵变化

在计算过程中，如果发现某次幂次矩阵已经不再变化，可以提前终止。

```
例如：题目26中，R³已经是全1矩阵
      那么R⁴、R⁵必然也是全1矩阵
      可以提前结束计算
```

#### 技巧2：利用对称性

如果关系具有某种对称性，可以利用这个性质简化计算。

#### 技巧3：逐步验证

每计算一个R^i，都检查是否有新的可达关系被发现：

```
R 包含：直达关系
R² 包含：2步可达关系
R³ 包含：3步可达关系
...
```

### 与题目26、27的联系

#### 题目26的特点

```
原始关系形成一个"连通分量"：
- a可以到b,c
- b可以到a,c
- c可以到a,b
- d可以到a（从而可以到b,c）
- e可以到d（从而可以到a,b,c）

由于连通性很好，传递闭包是全连通图（全1矩阵）
```

#### 题目27的特点

```
原始关系：
- e是"超级源点"：e可以到a,b,c,e
- 从e出发可以到达很多节点
- 通过a,b,c,d的相互连接，最终形成全连通图

同样，传递闭包是全连通图
```

### 验证答案的方法

对于传递闭包，可以通过以下方式验证：

1. **自反性检查**：如果R*是传递闭包，它不一定是自反的
   - 但如果图是强连通的，R*会包含所有自反对

2. **传递性检查**：如果(i,j)和(j,k)在R*中，那么(i,k)也应该在R*中

3. **最小性检查**：R*是包含R的最小传递关系
   - 不能移除任何有序对而不破坏传递性

### 总结

算法1（布尔矩阵乘法法）的核心逻辑：

1. **数学基础**：基于传递闭包的定义 R* = R ∪ R² ∪ ... ∪ Rⁿ
2. **计算方法**：通过布尔矩阵乘法计算各次幂，然后合并
3. **终止条件**：计算到Rⁿ即可（最长路径不超过n）
4. **效率分析**：时间复杂度O(n⁴)，空间复杂度O(n²）

虽然效率不如Warshall算法，但算法1提供了对传递闭包概念的深刻理解，是学习离散数学和算法的重要基础。
